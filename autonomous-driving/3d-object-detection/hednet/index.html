1. 研究背景与动机

1.1 点云 3D 检测的三条技术路线

从稀疏点云中学习有效表征，是 3D 目标检测的核心挑战。现有方法分三类：
point-based（点级）：用 PointNet 系列直接从原始点云提特征（如 PointRCNN、3DSSD）。需要昂贵的点采样和近邻搜索，计算开销大。
range-based（距离图像）：把点云投影成伪图像，复用成熟 2D 检测器。效率高但精度偏低。
voxel-based（体素）：把无序点云划分成规则体素，用稀疏 CNN 或 transformer 提特征。这是当前高性能 3D 检测的主流路线，HEDNet 也属于这一类。

1.2 核心痛点：submanifold 稀疏卷积阻断了信息交换

主流体素方法（SECOND、CenterPoint、TransFusion 等）的稀疏 backbone 主要由 SSR block（submanifold sparse residual block，子流形稀疏残差块） 堆叠而成，每个 SSR block 含两个小核（3×3×3）的 SS 卷积（submanifold sparse convolution，子流形稀疏卷积）。
SS 卷积的关键特性：只在 valid（非零）特征位置上计算，输出特征图与输入保持相同稀疏度。这带来效率，但也带来致命问题——它阻断了空间上不相连特征之间的信息交换。空间上离得远的两团点，永远无法通过堆叠 SS 卷积「看到」彼此，模型难以捕获 long-range dependency。
后果：对大目标（车的尺度是行人/骑车人的约 10 倍）和远距离目标（点稀疏、需要更多上下文）的检测精度受限。

1.4 灵感来源：编解码结构

作者回顾了各类稠密预测任务（语义分割、2D 检测）的 backbone 设计，发现 encoder-decoder（编解码）结构 在捕获 long-range dependency 上被反复验证有效：用高到低分辨率的编码器提多尺度特征，再用解码器恢复高分辨率特征以建模长程关系。例如 PSPNet 用金字塔池化聚合多尺度上下文，SWFormer 在 transformer backbone 里加 top-down 通路捕获跨窗口相关性。
但在 3D 稀疏卷积 backbone 中引入编解码结构，此前尚无人探索。HEDNet 正是填补这个空白。

2. 核心问题分析

2.1 一句话概括贡献

HEDNet 提出两种编解码积木，再用它们搭出一个层级网络：
SED block（Sparse Encoder-Decoder block，稀疏编解码块）：通过特征下采样把空间上离得远的 valid 特征拉到相邻位置实现信息交换，再通过多尺度融合恢复细节。关键特性——输入输出稀疏度保持一致，所以堆叠多个 SED block 不会引入过量计算。
DED block（Dense Encoder-Decoder block，稠密编解码块）：把 SED 的思路搬到 2D 稠密特征上，专门把稀疏特征向物体中心扩散，填补大目标中心的「空洞」（center-based 检测器依赖物体中心特征）。
HEDNet（Hierarchical Encoder-Decoder Network）：宏观上像 SECOND 一样层级下采样，微观上每个积木都是编解码结构——名字由此而来。

2.2 与既有路线的本质区别

相比 SS 卷积堆叠：SED block 用「下采样—融合」主动制造远距离特征交互，而不是被动等待感受野慢慢扩大。
相比 RS 卷积 / 大核：SED 通过 inverse conv 上采样时只恢复到原 valid 位置，稀疏度不膨胀，因此效率远高于 RSR block（实验里 RSR 176ms vs SED 48ms）。
相比 transformer（DSVT）：纯卷积实现，无需窗口注意力的串行计算，速度快 50%，精度还更高。

3. 技术架构详解

3.1 整体架构总览

给定原始点云，HEDNet 的前向流水线如下：
动态 VFE 体素化：用 DynamicVoxelVFE 把无序点云划成规则体素网格，记为 （分辨率 1×），输出体素特征。
3D 稀疏 backbone：由若干 SSR block + 若干 SED block 组成，逐级下采样提取 3D 稀疏特征。SED block 在这里负责捕获 long-range dependency。

3.2 SED block：稀疏编解码块（核心创新一）

先看基线 SSR block 的问题。 单个 SSR block 顺序作用两个 SS 卷积，输入输出间加 skip 连接。稀疏度定义为「未被 valid 特征占据的区域 / 特征图总面积」。SS 卷积只在 valid 特征上算，所以输出稀疏度与输入一致。代价是：空间上不相连的特征无法交换信息——下图（论文 Figure 1）里被星号标记的输出特征，收不到红框外那三个特征点（红三角）的信息。
RSR block（一种朴素补救）：把 SS 卷积换成 RS 卷积（regular sparse convolution，常规稀疏卷积）。RS 卷积同时作用在 valid 特征和 expanded 特征（valid 特征邻域内的位置）上，能捕获远距离上下文，但输出稀疏度比输入低。堆叠 RS 卷积会让稀疏度急剧下降、效率崩塌——这就是为什么已有方法只敢在下采样层用 RS 卷积。
下图对比了 SSR block (a)、RSR block (b) 和 HEDNet 的 SED block (c)：

SED block 的设计思想：通过特征下采样缩短远距离特征的空间间隔，再通过多尺度融合恢复下采样丢失的细节。
具体流程（以两尺度为例，对照上图 c）：
下采样后，底层特征图里空间上不相连的 valid 特征，被合并进中层特征图里相邻的 valid 特征——这一步实现了远距离信息交换。
在中层特征图上再作用一个 SSR block，促进 valid 特征间的交互。
最后把中层特征图上采样回输入分辨率。关键：上采样层（UP）只把特征恢复到输入特征图中 valid 特征覆盖的位置，因此 SED block 输入输出稀疏度严格一致，堆叠时不引入过量计算。

3.3 DED block：稠密编解码块（核心创新二）

动机：当前高性能 3D 检测器（CenterPoint、TransFusion、DSVT）大多依赖物体中心做检测。但纯稀疏 CNN 提取的特征图，在物体中心附近常有「空洞」——尤其大目标（车占很多体素，但中心可能没点）。中心没特征，center-based head 就难以定位。
DED block 的解法：把 SED 的编解码结构搬到 2D 稠密特征上，专门把稀疏特征向物体中心扩散，填补空洞。它与 SED 结构同构，只是把所有稀疏算子换成稠密算子：
SED 里的 SSR block → 换成 DR block（dense residual block，稠密残差块）（两个稠密 Conv2d）。
下采样的 RS 卷积 → 换成 stride=2 的 DR block。
上采样的 sparse inverse conv → 换成 dense deconvolution（稠密反卷积，ConvTranspose2d）。
因为是稠密卷积，特征自然向四周（含物体中心）扩散，刚好补上 center-based 检测所需的中心特征。下图是我按源码 DEDLayer.forward 重画的 DED block 结构：

3.4 HEDNet 整体组装

把 SED block 和 DED block 串起来就是 HEDNet：
3D 稀疏 backbone：dynamic VFE 体素化得到 → 一段 stem（subm 卷积 + 2 个 SSR block）→ 交替「SEDLayer + spconv 下采样」逐级降分辨率 → 末尾若干 SEDLayer。
压成 BEV：通过 transition 层沿 Z 轴用 stride 的 spconv 压扁，再 .dense() 把 reshape 成 的稠密 BEV。
2D 稠密 backbone： 个 DED block 串联（默认 Waymo 用 4 个、nuScenes 用 5 个），向中心扩散特征。
检测头：输出 spatial_features_2d 给 CenterHead / TransFusion-L head。
默认配置（Waymo）：SED/DED 都用三尺度、 ；voxel size m。

4. 数学公式与理论基础

4.1 稀疏度（sparsity）的定义

稀疏度是理解 HEDNet 效率的核心量。定义为：

SS 卷积：只在 valid 位置计算，输出稀疏度 = 输入稀疏度（不膨胀）→ 高效但阻断远距离信息。
RS 卷积：在 valid + expanded（邻域）位置计算，输出稀疏度 输入稀疏度（膨胀）→ 能捕获远距离上下文但越堆越稠密、越慢。

4.2 SED block 为什么能既扩感受野又保稀疏度

关键在「下采样 + 选择性上采样」这对操作：
下采样（Down） 用 RS 卷积，把空间上间隔 的两个 valid 特征，在 1/2 分辨率下间隔缩到 ，在 1/4 分辨率下缩到 。下采样后特征更密集、相互可见，从而在低分辨率层用普通 SSR block 就能完成远距离特征交互——等效于在原分辨率上获得了远超 3×3 核的感受野。
上采样（UP） 用 sparse inverse convolution，且只把特征写回输入特征图原有的 valid 位置（靠共享的 indice_key 记住下采样前的稀疏拓扑）。因此输出特征图的 valid 位置集合与输入完全一致：

这就是「堆叠多个 SED block 不会让稀疏度衰减、不引入过量计算」的理论根据，也是它相对 RSR block 的效率优势来源。

4.3 感受野的层级放大

宏观上 HEDNet 又在 SED block 外层套了一圈下采样（stem 的 spconv1、spconv2、spconv3）。于是感受野是两级放大：block 内编解码 + block 间层级下采样。这让网络在保持稀疏高效的同时，等效感受野足以覆盖大目标和远距离目标所需的上下文范围。

5.1 SEDLayer 的实现要点（编码器列表 + 解码器列表）

源码里 SEDLayer 用两个 nn.ModuleList 分别存编码器各级和解码器各级。前向时编码器逐级跑、把每级输出存进 feats；解码器从最低分辨率开始，每步 deconv 上采样后与对应编码器特征做 features 相加 + BN（即 skip 连接）。这正是公式 (4)(5) 的代码落点。

5.2 关键设计的代码体现：indice_key 共享

SED block 能「上采样只恢复到原 valid 位置」，靠的是下采样 spconv 和上采样 inverseconv 共享同一个 indice_key（源码里下采样用 f'spconv_{indice_key}_{idx}'，上采样 inverseconv 复用同名 key）。spconv 通过这个 key 记住下采样时的稀疏索引映射，逆卷积时严格按原拓扑还原，从而保证输入输出稀疏度一致。这是整个高效性的工程支点。

6. 核心创新点

把 HEDNet 的创新点拆开看，本质是三个层次：

6.1 创新一：SED block —— 用编解码让稀疏 CNN 捕获 long-range dependency

新在哪：首次把 encoder-decoder 结构引入 3D 稀疏卷积 backbone 的积木设计。此前编解码只用于稠密预测（FPN、PSPNet、UNet）或 transformer backbone（SWFormer）。
解决什么：SS 卷积阻断远距离信息交换的根本矛盾。
代码落点：SEDLayer（hednet.py），下采样用 SparseConv3d、上采样用 SparseInverseConv3d、残差用 SparseBasicBlock3D。
效果（Table 3a）：SED block 比 SSR block 高 1.96% L2 mAPH；比 RSR block 又快又准（48ms vs 176ms）。

6.2 创新二：DED block —— 向物体中心扩散特征

新在哪：把 SED 的编解码思路迁移到 2D 稠密域，专治大目标中心空洞。
解决什么：center-based 检测器（CenterPoint/TransFusion）依赖中心特征，而纯稀疏特征中心常缺失。
代码落点：DEDLayer + BasicBlock（稠密 Conv2d 残差，downsample=True 时 stride=2）。
效果（Table 3b）：加 DED block 后 L2 mAPH 从 69.89 提到 71.92（4 个最优，5 个过拟合）。

6.3 创新三：HEDNet —— 宏观层级 + 微观编解码的统一 backbone

新在哪：把 SED + DED 组织成一个层级网络，宏观像 SECOND、微观全是编解码。
效果：Waymo test 75.0% L2 mAPH、nuScenes test 72.0% NDS，且比 transformer 路线的 DSVT 快 50%。证明「纯卷积 + 编解码」可以同时拿下精度和效率，不必上 transformer。

6.4 泛化性：不止检测

附录 B 把 SED block 接到 MinkUNet34 上做 SemanticKITTI 语义分割，mIoU 比基线 +2.0%（68.3→70.3），说明 SED block 是通用的稀疏特征提取积木，不局限于检测任务。

7. 实验结果与分析

数据集：Waymo Open（160k/40k/30k train/val/test）、nuScenes（28k/6k/6k）。

8. 技术发展时间线

2018 SECOND：首次把 submanifold 稀疏卷积引入体素 3D 检测，确立「SSR block 堆叠 + RS 卷积下采样」的稀疏 backbone 范式。HEDNet 宏观结构沿用它。
2018 VoxelNet / 2019 PointPillars：体素/柱体编码奠基。
2021 CenterPoint：center-based 检测头成为主流，催生了「物体中心特征很重要」这一前提——这正是 DED block 要解决的问题。
2022 SST / 2022 SWFormer / 2023 DSVT：transformer 路线尝试捕获长程依赖；SWFormer 首次在稀疏 backbone 加 top-down 通路（编解码雏形）。DSVT 是 HEDNet 之前的 SOTA。
2022 LargeKernel3D / 2023 LinK：大核稀疏卷积路线，靠大核扩感受野但低效。
2023 HEDNet（本篇，NeurIPS）：用编解码积木（SED/DED）让纯稀疏 CNN 高效捕获长程依赖，同时超越大核与 transformer 路线。
2024 SAFDNet（CVPR，同作者续作）：在 HEDNet 基础上做全稀疏（fully sparse）检测——HEDNet 末段仍要转稠密 BEV，SAFDNet 用 adaptive feature diffusion（AFD）在稀疏域直接向中心扩散，省掉稠密 backbone，更适合大范围场景（如 Argoverse2）。同仓库已含其实现（SparseHEDNet 类）。

9.2 SSR block / SS 卷积的代码落点

SparseBasicBlock3D 就是论文的 SSR block：两个 3×3×3 的 SubMConv3d（submanifold 稀疏卷积）+ BN + 残差。注意 replace_feature——spconv 张量的 features 是单独张量，BN/ReLU/相加都要通过它替换：
class SparseBasicBlock3D(spconv.SparseModule):
    def __init__(self, dim, indice_key, norm_fn=norm_fn_1d, bias=True):
        super().__init__()
        # 两个 submanifold 稀疏卷积，共享 indice_key → 输入输出稀疏拓扑一致
        self.conv1 = spconv.SubMConv3d(dim, dim, 3, 1, 1, bias=bias, indice_key=indice_key)
        self.bn1 = norm_fn(dim)
        self.conv2 = spconv.SubMConv3d(dim, dim, 3, 1, 1, bias=bias, indice_key=indice_key)
        self.bn2 = norm_fn(dim)
        self.relu = nn.ReLU()
    def forward(self, x):
        out = self.conv1(x)                                   # SS 卷积只在 valid 位置算
        out = out.replace_feature(self.bn1(out.features))
        out = out.replace_feature(self.relu(out.features))
        out = self.conv2(out)
        out = out.replace_feature(self.bn2(out.features))
        out = out.replace_feature(self.relu(out.features + x.features))  # 残差
        return out

9.3 SEDLayer 源码逐段拆解（核心）

SEDLayer 把编码器、解码器各存成一个 ModuleList，xy_only 开关决定用 2D 还是 3D 稀疏卷积（这是同一套代码同时支持 3D backbone 和 2D 单步长 backbone 的关键）：
class SEDLayer(spconv.SparseModule):
    def __init__(self, dim, down_kernel_size, down_stride, num_SBB, indice_key, xy_only=False):
        super().__init__()
        # xy_only=True → 2D 稀疏卷积（用于 nuScenes 单步长版 / SAFDNet 的 AFD 层）
        block = SparseBasicBlock2D if xy_only else SparseBasicBlock3D
        post_act_block = post_act_block_sparse_2d if xy_only else post_act_block_sparse_3d
        # ---- 编码器 ----
        # 第 0 级：num_SBB[0] 个 SSR block（不下采样），对应公式 F1 = SSR^m(X)
        self.encoder = nn.ModuleList([
            spconv.SparseSequential(
                *[block(dim, indice_key=f"{indice_key}_0") for _ in range(num_SBB[0])])
        ])
        num_levels = len(down_stride)
        for idx in range(1, num_levels):
            cur_layers = [
                # 下采样：stride 的 spconv，indice_key 命名 spconv_{key}_{idx}（关键！上采样要复用）
                post_act_block(dim, dim, down_kernel_size[idx], down_stride[idx],
                               down_kernel_size[idx] // 2,
                               conv_type='spconv', indice_key=f'spconv_{indice_key}_{idx}'),
                # 下采样后再跑 num_SBB[idx] 个 SSR block（在更密集的低分辨率上做特征交互）
                *[block(dim, indice_key=f"{indice_key}_{idx}") for _ in range(num_SBB[idx])]
            ]
            self.encoder.append(spconv.SparseSequential(*cur_layers))
        # ---- 解码器 ----
        # 从最低分辨率往回走，每级一个 inverseconv（复用编码器同名 indice_key → 只还原到原 valid 位置）
        self.decoder = nn.ModuleList()
        self.decoder_norm = nn.ModuleList()
        for idx in range(num_levels - 1, 0, -1):
            self.decoder.append(
                post_act_block(dim, dim, down_kernel_size[idx],
                               conv_type='inverseconv', indice_key=f'spconv_{indice_key}_{idx}'))
            self.decoder_norm.append(norm_fn_1d(dim))
    def forward(self, x):
        feats = []
        for conv in self.encoder:          # 编码器逐级前向，保存每级特征
            x = conv(x)
            feats.append(x)                # feats = [F1, F2, F3]（三尺度时）
        x = feats[-1]                      # 从最低分辨率 F3 开始解码
        # zip: 解码器 / norm / 编码器特征逆序(去掉最后一个)，即 [F2, F1]
        for deconv, norm, up_x in zip(self.decoder, self.decoder_norm, feats[:-1][::-1]):
            x = deconv(x)                                       # 上采样（只到原 valid 位置）
            x = replace_feature(x, norm(x.features + up_x.features))  # skip 相加 + BN → F4, F5
        return x
逐段对照公式：
编码器第 0 级 →  
编码器第 1、2 级（含 Down）→  
解码器第 1 步 →  
解码器第 2 步 →  
张量稀疏度（3D，dim=64 时）：输入   是 SparseConvTensor，features 形状  。下采样后   减少（特征更密集）；上采样靠 indice_key='spconv_sedlayerX_idx' 还原回下采样前的 indices，所以输出的  、indices 与输入完全一致——这就是「稀疏度不变」在代码里的保证。
默认超参（Waymo cfg）：SED_NUM_SBB=[2,1,1]（三级各 2/1/1 个 SSR block）、SED_DOWN_STRIDE=[1,2,2]（第 0 级不降，后两级各降 2 倍）、SED_DOWN_KERNEL_SIZE=[3,3,3]、SED_FEATURE_DIM=64、SED_NUM_LAYERS=2（主段 2 个 SEDLayer）。down_stride[0]==1 有 assert 保证第 0 级不下采样。

9.4 DEDLayer 源码：与 SED 同构但全稠密

DEDLayer 结构与 SEDLayer 一一对应，只是把稀疏算子换成稠密算子，不再需要 indice_key（稠密卷积没有稀疏拓扑问题）：
class DEDLayer(nn.Module):
    def __init__(self, dim, down_stride, num_SBB):
        super().__init__()
        # 编码器第 0 级：num_SBB[0] 个稠密残差块 BasicBlock（= DR block）
        self.encoder = nn.ModuleList(
            nn.Sequential(*[BasicBlock(dim) for _ in range(num_SBB[0])]))
        num_levels = len(down_stride)
        for idx in range(1, num_levels):
            # 下采样用 stride=2 的 DR block（BasicBlock(downsample=True)）
            cur_layers = [BasicBlock(dim, downsample=True)]
            cur_layers.extend([BasicBlock(dim) for _ in range(num_SBB[idx])])
            self.encoder.append(nn.Sequential(*cur_layers))
        self.decoder = nn.ModuleList()
        self.decoder_norm = nn.ModuleList()
        for idx in range(num_levels - 1, 0, -1):
            # 上采样用稠密反卷积 ConvTranspose2d
            self.decoder.append(
                post_act_block_dense_2d(dim, dim, down_stride[idx], down_stride[idx], 0, conv_type='deconv'))
            self.decoder_norm.append(norm_fn_2d(dim))
    def forward(self, x):
        feats = []
        for conv in self.encoder:
            x = conv(x); feats.append(x)
        for deconv, norm, up_x in zip(self.decoder, self.decoder_norm, feats[:-1][::-1]):
            x = norm(deconv(x) + up_x)   # 稠密相加 + BN（普通张量直接加，无需 replace_feature）
        return x
其中 BasicBlock（DR block）的下采样靠 conv1 的 stride=1+int(downsample)，并配一条 1×1 stride2 的 downsample_layer 对齐残差维度——标准 ResNet 残差块写法。

9.5 HEDNet.forward：3D 稀疏 → BEV → 2D 稠密 全链路

def forward(self, batch_dict):
    x = spconv.SparseConvTensor(                 # 用 VFE 输出构造稀疏张量
        features=batch_dict['voxel_features'],
        indices=batch_dict['voxel_coords'].int(),
        spatial_shape=self.sparse_shape,         # grid_size[::-1] + [1,0,0]
        batch_size=batch_dict['batch_size'])
    x = self.stem(x)                             # 3D: subm + 2×SSR + 交替(SEDLayer + spconv 下采样)
    for layer in self.sed_layers:                # 主 SED 段（默认 2 个）
        x = layer(x)
    x = self.to_bev(x)                           # transition 沿 Z 压扁 + .dense() + reshape 成 BEV
    for layer in self.ded_layers:                # 2D DED 段（默认 4 个）
        x = layer(x)
    batch_dict.update({'spatial_features_2d': x})
    return batch_dict
def to_bev(self, x):
    x = self.transition(x).dense()               # (N, C, D, H, W)
    N, C, D, H, W = x.shape
    x = x.view(N, C * D, H, W)                    # 把 Z 维并入通道 → BEV (N, C*D, H, W)
    return x
stem 里 SED block 与下采样 spconv 是交错写死在 SparseSequential 里的（见 §3.1 数据流），这是「层级（hierarchical）」的宏观体现；to_bev 的 transition 用两个 (3,1,1) stride (2,1,1) 的 spconv 专门压 Z 轴高度。

9.6 续作 SAFDNet 的 AFD（顺带，理解全稀疏方向）

同文件的 SparseHEDNet 实现了 CVPR 2024 续作。它把 HEDNet 的「转稠密 BEV + DED 稠密 backbone」换成全稀疏：用 feature_diffusion（adaptive feature diffusion，AFD）在稀疏域预测每个体素是否落在某类物体框内（cls_conv + focal loss 监督），只对前景体素做 max_pool 式扩散填补中心空洞，再用 xy_only=True 的 SEDLayer（afd_layers）继续提特征。这避免了大范围场景（Argoverse2 等）转稠密的显存/算力爆炸——是 HEDNet 在「全稀疏」方向的自然延伸。马爷若关注大范围 occ，这条线值得留意。

10. 局限性与未来方向

仅面向室外自动驾驶场景：论文明确指出 HEDNet 主要针对室外点云检测，在室内场景的适用性仍是开放问题（室内点云密度分布与室外差异大）。
末段仍需转稠密 BEV：HEDNet 的 DED 段在稠密域运行，大范围场景下稠密 backbone 的显存/算力会显著上升。续作 SAFDNet 用全稀疏 AFD 缓解了这点。
训练增益与方法增益耦合：Table 5 显示很大一块涨点来自 DSVT 的训练 tricks（IoU loss、faded、class-specific NMS）。复现时若不带这些 tricks，SED/DED 的净增益会被低估。
超参敏感：DED block 数量过多（5 个）会过拟合；尺度数增加精度边际递减但延迟线性上升。需按场景调。
未来方向：全稀疏化（已由 SAFDNet 推进）、跨任务通用稀疏积木（已在分割上验证）、与多模态/时序融合结合。

11. 总结

HEDNet 的核心贡献，是把成熟的 encoder-decoder 思想重新带回 3D 稀疏卷积 backbone 的积木级设计，用两个简单而有效的积木解决了点云检测的两个老问题：
SED block 用「下采样拉近远距离特征 + 选择性上采样保稀疏度」让纯稀疏 CNN 高效捕获 long-range dependency，效率远超 RSR/大核，精度超过 transformer。
DED block 用稠密编解码向物体中心扩散特征，补上 center-based 检测器最需要的中心特征。
二者组装成 HEDNet，在 Waymo（75.0 L2 mAPH）和 nuScenes（72.0 NDS）双双刷新 SOTA，且比 DSVT 快 50%；SED block 还能泛化到语义分割。
一句话：不靠 transformer、不靠大核，纯卷积 + 编解码，把稀疏 backbone 的精度和效率同时拉满。

11.1 对马爷 E2E Occupancy 项目 LiDAR 分支的工程启发 ⭐

这篇是收官论文里和马爷 occ 项目工程最贴近的一篇（LiDAR 分支通常就是 SECOND + 稀疏卷积 backbone）。可落地的启发：
直接升级 LiDAR backbone：occ 项目若用 SECOND/SSR 堆叠的稀疏 backbone，可把若干 SSR 段替换成 SEDLayer。代码改动小（OpenPCDet 风格，SEDLayer 即插即用），实测在检测上 +1.6%~2% L2 mAPH，对大目标和远距离 voxel 增益最大——而 occ 恰恰最难的就是远处和大物体的占据预测。
占据预测天然受益于「向中心扩散」：DED block 填补大目标中心空洞的思路，对 occ 的「物体内部体素该不该占据」判断直接相关。可借鉴 DED 的稠密编解码做 occ decoder 的特征致密化。
稀疏度守恒 = 工程可控的算力：SED 的「上采样只回原 valid 位置」保证堆叠不爆算力，这对 occ 这种本就稠密、算力紧张的任务尤其重要——可在不显著增加 latency 的前提下扩感受野。
全稀疏方向（SAFDNet/AFD）：若 occ 走大范围（如 Argoverse2 尺度或更大 BEV 范围），可参考续作的 AFD 在稀疏域扩散、避免转稠密，省显存。
复现注意：迁移时要连带 DSVT 训练配方（IoU loss、faded、class-specific NMS、weight decay 0.05）一起上，否则净增益会被埋没（Table 5 教训）。
感受野是 occ 的隐形瓶颈：HEDNet 用「block 内编解码 + block 间层级下采样」两级放大感受野的做法，比单纯加深或加大核更划算，值得在 occ backbone 设计里直接套用。
