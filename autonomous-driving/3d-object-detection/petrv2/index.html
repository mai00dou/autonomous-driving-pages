谱系定位：本文是 PETR 谱系战役第 5 篇。DETR → DETR3D（query 侧 ref point）→ PETR（3D PE 隐式，feature 侧）→ BEVFormer（显式 BEV grid + SCA + TSA）→ BEVDet4D（LSS + 时序 align+concat）→ PETRv2（PETR 的时序版）。它把 PETR 的 3D position embedding 扩展到时序，引入 feature-guided position encoder（FPE），并统一到 3D 检测 + BEV 分割 + 3D 车道线多任务。

1. 研究背景与动机

多相机 3D 感知是纯视觉自动驾驶的核心。当时的多视图 3D 检测方法分两大流派：
BEV-based（如 BEVDet、BEVFormer、BEVDet4D）：用 LSS 或 BEV query 把多视图特征显式投影成鸟瞰图（BEV）表征，再在 BEV 上做检测/分割。
DETR-based（如 DETR3D、PETR）：把每个 3D 物体建模成一个 object query，用匈牙利匹配做端到端建模。其中 PETR 把多视图 2D 特征通过加上 3D position embedding（3D PE） 转换成「3D 位置感知特征」，query 从 3D 空间初始化后直接与这些特征交互。
PETR-v1 虽然优雅，但有三个明显短板，正是 PETRv2 要补的：
没有时序。单帧检测对速度估计、遮挡恢复无能为力。BEVFormer 用 temporal self-attention、BEVDet4D 用 BEV 特征对齐+concat 引入时序，都是在 BEV 特征层面 做对齐。但 PETR 把 3D 位置隐式编码进 2D 特征里，无法做显式 BEV 特征变换。作者提出疑问：3D PE 这套隐式表征，能不能直接用来做时序对齐？
3D PE 与图像无关（data-independent）。PETR 的 3D PE 只由相机视锥 meshgrid 经相机参数生成，同一相机所有图像共享同一份 3D PE，没有利用图像内容（如深度线索）。
只能做检测。BEVFormer 用 BEV query 可统一检测+分割，但当 BEV 分辨率大时 BEV query 数量爆炸（256×256 ＞ 60000 个），不适合 PETR 的全局注意力 decoder。
PETRv2 的三个贡献正好对应：
把位置编码变换扩展到 时序表征学习——时序对齐通过对 3D PE 做位姿变换实现；并提出 FPE 用 2D 特征引导重加权 3D PE。
一套简洁有效的 多任务方案：BEV 分割、3D 车道线通过引入「任务专属稀疏 query」支持。
在 nuScenes 3D 检测、BEV 分割、OpenLane 3D 车道线三个任务上达到 SOTA，并对 PETR 框架做了详尽的鲁棒性分析（外参噪声、相机缺失、时延）。

2. 核心问题分析

2.1 时序对齐：在哪个层面对齐？

时序建模的核心问题是 如何把不同帧的物体位置在 3D 空间对齐。三种代表方案对比：
BEVDet4D：显式把前一帧的 BEV 特征经位姿变换对齐到当前帧，再 concat → 在 BEV 特征层面 对齐。
BEVFormer：用 temporal self-attention（TSA）让当前 BEV query 去 attend 历史 BEV 特征 → 也是 BEV 特征层面，但用注意力软融合。
PETRv2（本文）：PETR 没有显式 BEV 特征，3D 位置是隐式编码在 2D 特征里的。作者发现：只要把前一帧的 3D 坐标经 ego-motion 对齐到当前帧坐标系，再生成 3D PE，PETR 就能很好地工作。即在 隐式 3D PE 层面 做时序对齐，完全不需要显式 BEV 特征变换。
这是 PETRv2 最关键的洞察：时序对齐的本质是「对齐物体的 3D 位置」，而 PETR 的 3D PE 恰恰编码的就是 3D 位置，所以对齐 3D 坐标 = 对齐 3D PE = 对齐物体位置。工程上这意味着 不需要构建 BEV 网格、不需要 grid sampling，对齐只是一次 4×4 矩阵乘法。

2.2 3D PE 的数据无关性

PETR 的 3D PE 由固定视锥 meshgrid 生成，与图像内容无关。作者认为 3D PE 应当由 2D 特征驱动，因为图像特征能提供有用线索（如深度）。于是用 FPE 把 2D 特征作为 attention 权重去重加权 3D PE，让其变成 data-dependent。

2.3 多任务的 query 定义

BEVFormer 的稠密 BEV query 数量太大，不适合 PETR 全局注意力。PETRv2 改用 稀疏的任务专属 query：检测 query 定义在整个 3D 空间，分割 query 在 BEV 空间按 patch 初始化，车道 query 用 3D anchor lane 表示。三种 query 进同一个 transformer decoder，再各自接任务头。

3. 技术架构详解

3.1 整体架构（Overall Architecture）

PETRv2 整体流程（论文 Fig. 1）：
2D 特征提取：当前帧 t 与前帧 t-1 的多视图图像分别送入共享的 2D backbone（ResNet-50 / VoVNetV2 / EfficientNet）+ FPN，取 P4 特征（合并 C4、C5，1/16 分辨率）作为 2D 特征。
3D 坐标生成：与 PETR 一致，从相机视锥 meshgrid 经相机内外参反投影到 lidar 3D 坐标。
时序对齐（A）：考虑 ego motion，前帧 t-1 的 3D 坐标先经位姿变换对齐到当前帧 t 的坐标系。
拼接（C）：两帧的 2D 特征、3D 坐标分别沿相机（batch）轴 concat，相当于把 12 路（6+6）特征喂给后续模块。
FPE：拼接后的 2D 特征 + 3D 坐标送入 feature-guided position encoder，生成 transformer decoder 用的 key/value。
多任务 query：检测 query、分割 query、车道 query 从各自空间初始化，进入同一个 transformer decoder 与 key/value 交互。
任务头：更新后的 query 分别送入检测头、分割头、车道头，输出 3D 框、BEV 分割图、3D 车道线。

时序对齐的代码落点
PETRv2 一个工程巧思：3D 坐标对齐不在模型 forward 里做，而是在数据 pipeline 里预先算好 lidar2img。前帧（sweep）加载时，其 lidar2img 已经是「前帧像素 → 当前帧 lidar 坐标系」的复合矩阵，所以模型里 position_embeding() 对当前帧和前帧用同一套逻辑反投影，得到的就是已对齐到当前帧坐标系的 3D 坐标。
nuscenes_dataset.py 的 get_data_info 预计算每个相机（含 sweep）的 lidar2img：
# nuscenes_dataset.py — get_data_info()
for cam_type, cam_info in info['cams'].items():
    image_paths.append(cam_info['data_path'])
    # sensor2lidar：相机到 lidar 的旋转/平移（标注里已对齐到关键帧 lidar）
    lidar2cam_r = np.linalg.inv(cam_info['sensor2lidar_rotation'])
    lidar2cam_t = cam_info['sensor2lidar_translation'] @ lidar2cam_r.T
    lidar2cam_rt = np.eye(4)
    lidar2cam_rt[:3, :3] = lidar2cam_r.T
    lidar2cam_rt[3, :3] = -lidar2cam_t
    intrinsic = cam_info['cam_intrinsic']
    viewpad = np.eye(4); viewpad[:intrinsic.shape[0], :intrinsic.shape[1]] = intrinsic
    lidar2img_rt = (viewpad @ lidar2cam_rt.T)   # lidar → 像素
    extrinsics.append(lidar2cam_rt)
    lidar2img_rts.append(lidar2img_rt)

3.3 多任务学习（Multi-task Learning）

PETRv2 给 PETR 加上 seg query 和 lane query 来支持 BEV 分割与 3D 车道线，三类 query 共享同一个 backbone + 3D PE + transformer decoder，只在 query 初始化空间和任务头上不同。
检测 query（det query）：定义在整个 3D 空间，用一组可学习 anchor point 经小 MLP 生成（与 PETR 一致）。
分割 query（seg query）：高分辨率 BEV 图被切成若干 patch，每个 seg query 对应一个 patch（如 BEV 图左上 25×25 像素）。seg query 用 BEV 空间固定 anchor point 初始化，经两层 MLP 投影成 query，进 decoder 后再送分割头（类似 CVT 的 decoder 上采样到最终 BEV 分割图），用 focal loss 逐类监督。论文设置：200×200 BEV ground truth，patch 25×25，共 625 个 seg query。
车道 query（lane query）：3D anchor lane 表示，每条 anchor lane 是一组有序 3D 坐标 。与 Persformer 不同，PETRv2 的 anchor lane 平行于 Y 轴（Persformer 给每条 anchor 预设不同斜率）。车道头预测车道类别 C、相对 anchor 的 x/z 偏移 ，以及长度可变的可见性向量 （控制起止点）。类别和可见性用 focal loss，偏移用 L1 loss。
下图（论文 Fig. 3）展示三类 query 的定义空间：det query 在整个 3D 空间，seg query 在 BEV 空间，lane query 是 3D anchor line（300 个 anchor point 构成）。

分割头是一组上采样卷积块（DecoderBlock：Upsample×2 → Conv3×3 → ReLU → Conv1×1，带残差 skip），把 BEV query reshape 成特征图后逐级上采样到 200×200 输出：
# petr_head_seg.py — 分割上采样解码器
class DecoderBlock(nn.Module):
    def __init__(self, in_channels, out_channels, skip_dim, residual, factor):
        dim = out_channels // factor
        self.conv = nn.Sequential(
            nn.Upsample(scale_factor=2, mode='bilinear', align_corners=True),
            nn.Conv2d(in_channels, dim, 3, padding=1, bias=False),
            nn.ReLU(inplace=True),
            nn.Conv2d(dim, out_channels, 1, padding=0, bias=False))
        self.up = nn.Conv2d(skip_dim, out_channels, 1) if residual else None
    def forward(self, x, skip):
        x = self.conv(x)
        if self.up is not None:
            x = x + F.interpolate(self.up(skip), x.shape[-2:])
        return self.relu(x)

FPE 的代码落点
FPE 在代码里就是一个 SELayer（Squeeze-Excitation 风格的门控），这是理解 FPE 最直接的角度——它不是把特征加进去，而是用特征生成 逐元素门控权重 去调制 3D PE。
 （门控分支）的实现 SELayer：
# petrv2_head.py — FPE 的门控分支 ξ
class SELayer(nn.Module):
    def __init__(self, channels, act_layer=nn.ReLU, gate_layer=nn.Sigmoid):
        super().__init__()
        self.conv_reduce = nn.Conv2d(channels, channels, 1, bias=True)   # 1×1
        self.act1  = act_layer()
        self.conv_expand = nn.Conv2d(channels, channels, 1, bias=True)   # 1×1
        self.gate  = gate_layer()                                        # Sigmoid
    def forward(self, x, x_se):
        # x = ψ(3D 坐标) 生成的 3D PE；x_se = 2D 图像特征
        x_se = self.conv_reduce(x_se)
        x_se = self.act1(x_se)
        x_se = self.conv_expand(x_se)
        return x * self.gate(x_se)        # ★ 逐元素相乘 = ξ(F) ∗ ψ(P)
 （位置编码分支）是 position_encoder，把 inverse_sigmoid 后的 3D 坐标经 Conv-ReLU-Conv 升到 embed_dims：
# petrv2_head.py — _init_layers()
self.position_encoder = nn.Sequential(            # ψ
    nn.Conv2d(self.position_dim, self.embed_dims*4, 1, 1, 0),
    nn.ReLU(),
    nn.Conv2d(self.embed_dims*4, self.embed_dims, 1, 1, 0))
if self.with_fpe:
    self.fpe = SELayer(self.embed_dims)           # ξ 门控
3D 坐标生成 position_embeding()（与 PETR 同款，含 LID 深度离散化）：
# petrv2_head.py — position_embeding()
# 1) 视锥深度离散：LID（线性递增离散）或均匀
if self.LID:
    index_1 = index + 1
    bin_size = (self.position_range[3] - self.depth_start) / (self.depth_num*(1+self.depth_num))
    coords_d = self.depth_start + bin_size * index * index_1   # 近密远疏
else:
    bin_size = (self.position_range[3] - self.depth_start) / self.depth_num
    coords_d = self.depth_start + bin_size * index
# 2) 视锥 meshgrid (W,H,D,4)，齐次化，乘深度
coords = torch.stack(torch.meshgrid([coords_w, coords_h, coords_d])).permute(1,2,3,0)
coords = torch.cat((coords, torch.ones_like(coords[..., :1])), -1)
coords[..., :2] = coords[..., :2] * torch.maximum(coords[..., 2:3], ones*eps)
# 3) img2lidar = inv(lidar2img)，把视锥点反投影到 lidar 3D 坐标
img2lidars = inv(img_meta['lidar2img'])           # ★ 前帧的 lidar2img 已是对齐到当前帧的
coords3d = torch.matmul(img2lidars, coords).squeeze(-1)[..., :3]
# 4) 归一化到 [0,1]（按 position_range），无效点 mask，inverse_sigmoid
coords3d[...,0:1] = (coords3d[...,0:1]-range[0]) / (range[3]-range[0])  # x，y、z 同理
coords3d = inverse_sigmoid(coords3d)
coords_position_embeding = self.position_encoder(coords3d)             # ψ(P)
forward 里 FPE 的调用一行到位（先 ψ 生成 3D PE，再用 2D 特征 x 门控）：
# petrv2_head.py — forward()
coords_position_embeding, _ = self.position_embeding(mlvl_feats, img_metas, masks)  # ψ(P)
if self.with_fpe:
    coords_position_embeding = self.fpe(
        coords_position_embeding.flatten(0,1), x.flatten(0,1)).view(x.size())        # ξ(F)∗ψ(P)
pos_embed = coords_position_embeding
# 再叠加正弦多视图 PE
sin_embed = self.adapt_pos3d(self.positional_encoding(masks).flatten(0,1)).view(x.size())
pos_embed = pos_embed + sin_embed

3.5 Transformer Decoder 与 query 更新

检测分支的 decoder 调用与 query 更新（PETR 风格的全局 cross-attention，无 deformable、无显式 BEV）：
# petrv2_head.py — forward()
reference_points = self.reference_points.weight                       # 可学习 anchor (num_query,3)
query_embeds = self.query_embedding(pos2posemb3d(reference_points))    # 3D 正弦编码 → MLP
reference_points = reference_points.unsqueeze(0).repeat(batch_size,1,1)
outs_dec, _ = self.transformer(x, masks, query_embeds, pos_embed, self.reg_branches)
outs_dec = torch.nan_to_num(outs_dec)
# 逐 decoder 层：分类 + 回归（xy/z 相对 reference 残差 → sigmoid → 反归一到 pc_range）
for lvl in range(outs_dec.shape[0]):
    reference = inverse_sigmoid(reference_points.clone())
    outputs_class = self.cls_branches[lvl](outs_dec[lvl])
    tmp = self.reg_branches[lvl](outs_dec[lvl])
    tmp[..., 0:2] += reference[..., 0:2]; tmp[..., 0:2] = tmp[..., 0:2].sigmoid()
    tmp[..., 4:5] += reference[..., 2:3]; tmp[..., 4:5] = tmp[..., 4:5].sigmoid()
    if self.with_time:
        tmp[..., 8:] = tmp[..., 8:] / mean_time_stamp[:, None, None]   # 速度归一
注意 key/value 是 12 路（两帧×6 相机）特征，但 query 数量不变（900/1500），decoder 通过全局注意力自动从两帧特征里聚合时序信息。这与 BEVFormer 的 TSA（query 数 = BEV grid）形成鲜明对比。

3.6 鲁棒性分析的实验设定

PETRv2 对 PETR 框架做了系统鲁棒性分析（论文 Fig. 5），模拟三类常见传感器误差：
外参噪声（extrinsics noise）：车辆颠簸或环境力导致相机抖动/偏移，系统提供的外参不准。实验中随机选一个相机施加 3D 旋转，记 表示三轴最大旋转角（度）均为 M。
相机缺失（camera miss）：某个相机损坏或被遮挡。多视图本应提供全景，但现实中可能缺一路。
相机时延（camera time delay）：曝光时间（尤其夜间长曝光）导致喂入系统的是前一时刻图像，带来显著输出偏移。时延单位 （两 sweep 间隔）。

5. 详细流程图

完整的端到端数据流（结合 §3 的三张自绘图）：
关键帧 t + 前帧 t-1（sweep_range [3T, 27T] 内采样，推理固定 15T）的 6 路图像 → 共享 backbone+FPN → P4 特征。
数据 pipeline 预算好两帧各相机的 lidar2img：前帧的 lidar2img 已经把「前帧像素→当前帧 lidar」复合进去（含 ego-motion），所以模型侧无需再做对齐矩阵乘。
position_embeding() 对两帧统一反投影出 3D 坐标（前帧的天然已对齐），LID 深度离散，归一化 + inverse_sigmoid。
ψ（position_encoder）把 3D 坐标编码成 3D PE；ξ（SELayer）用 2D 特征生成 Sigmoid 门控，逐元素调制 3D PE（FPE）。
两帧特征/PE 沿 batch 轴 concat（12 路），3D PE + 2D 特征 = key，2D 特征 = value。
det/seg/lane query 从各自空间初始化 → PETR transformer decoder 全局 cross-attention 聚合两帧信息。
速度回归除以两帧 Δt 还原真实速度；各任务头输出 3D 框 / BEV 分割 / 3D 车道。

6. 核心创新点

PETRv2 相对 PETR-v1 的三个增量，是理解本文最重要的脉络：
创新 1：时序 3D PE 对齐（Temporal 3D PE Alignment / CA）
把 PETR 的 3D 位置编码扩展到时序——前帧 3D 坐标经 ego-motion 位姿变换对齐到当前帧坐标系，再生成 3D PE 参与当前帧。代码落点：数据 pipeline 预算前帧 lidar2img（已含对齐），模型侧 position_embeding 统一反投影，速度在 with_time 分支除以 Δt。意义：在 隐式 3D PE 层面 实现时序，不需显式 BEV 特征变换。直接收益是速度估计大幅改善（mAVE 0.808 → 0.343 m/s）。
创新 2：特征引导位置编码器（FPE）
用 2D 图像特征（含深度线索）生成 Sigmoid 门控，逐元素重加权 3D PE，让 3D PE 从 data-independent 变成 data-dependent。代码落点：SELayer（ξ 门控）+ position_encoder（ψ 编码）。意义：3D PE 不再只由几何决定，而是被视觉先验调制，提升数据适应性与外参噪声鲁棒性。
创新 3：多任务统一（稀疏任务专属 query）
检测/分割/车道用稀疏的任务专属 query（分别在 3D / BEV / 3D anchor lane 空间初始化），共享 backbone + 3D PE + decoder，避免 BEVFormer 稠密 BEV query 的数量爆炸。意义：一套框架同时做 3D 检测、BEV 分割、3D 车道线，且 query 稀疏、计算可控。

8. 技术发展时间线

PETR 谱系（多相机 3D 感知）的演进脉络：
DETR（2020）：集合预测 + 匈牙利匹配，开启端到端检测，但是 2D。
DETR3D（2021）：把 object query 放到 3D 空间，query 投影回 2D 图采样特征（query 侧 reference point，稀疏采样）。
PETR（2022）：反过来——不让 query 去采样，而是给 2D 特征加 3D position embedding 变成「3D 位置感知特征」，query 在 3D 空间用全局注意力直接交互（feature 侧隐式 3D）。
BEVFormer（2022）：显式 BEV grid query + 空间 cross-attention（SCA）聚合多视图 + 时序 self-attention（TSA）融合历史 BEV。
BEVDet4D（2022）：BEVDet（LSS 显式投影 BEV）+ 时序，把前帧 BEV 特征经位姿对齐后 concat。
PETRv2（2022，本文）：PETR 的时序版。把 3D PE 扩展到时序（坐标对齐），引入 FPE，统一多任务。与 BEVFormer/BEVDet4D 不同，它在隐式 3D PE 层面对齐，不建 BEV。
后续：StreamPETR（以 object query 做长时序传播）、PETR 系思想延伸到 occupancy、端到端等。
PETRv2 在这条线上的意义：证明了「隐式 3D 位置编码」这套范式也能优雅地吃下时序和多任务，不必非走显式 BEV 那条更重的路。

9. 代码实现技巧

除了 §3 已拆的核心 forward，PETRv2 仓库里还有几个对复现/落地很关键的工程技巧：
(1) 时序对齐前移到 dataloader。最重要的工程决策：3D 坐标对齐不在 GPU forward 里做，而在 CPU 数据 pipeline 预算 lidar2img。前帧 sweep 的 lidar2img 已是「前帧像素 → 当前帧 lidar」的复合矩阵，模型对两帧用同一套反投影逻辑即可。好处：forward 无额外矩阵乘、无需传 ego-pose 进网络、增广与几何天然同步。
(2) query denoise（DN-DETR 扩展）。petrv2_dnhead.py 把 DN-DETR 的去噪 query 引入 3D 检测加速收敛：对每个 GT 框中心加随机噪声（小于 (w/2, l/2, h/2)），让模型学会从带噪 anchor 恢复真值。配置在 configs/denoise/。
(3) LID 深度离散。LID=True 时深度 bin 用「线性递增离散」（near 密 far 疏）：coords_d = depth_start + bin_size·index·(index+1)，比均匀离散更符合「近处需要高分辨率」的几何直觉。
(4) 前帧特征 detach（分割任务）。petr_head_seg.py 的 with_detach：前帧特征 past_frame.detach() 只提供时序线索、不回传梯度，省显存并稳定训练。
(5) GridMask 数据增广。petr3d.py 初始化 GridMask(True, True, rotate=1, ratio=0.5, mode=1, prob=0.7)，训练时随机遮挡图像网格区域，防过拟合。
(6) CBGS + 多尺度。检测主榜用 CBGS（类别平衡采样）和多尺度输入（800×320 + 1600×640，即 test 表的 "ms"），换取 mAP/NDS 进一步提升。
(7) 速度归一化与坐标反归一。回归输出 xy/z 是相对 reference 的残差经 sigmoid 归一，最后按 pc_range 反归一到真实米制；速度分量除以两帧 Δt（with_time）。
(8) 分组回归头 RegLayer。group_reg_dims=(2,1,3,2,2) 对应 (xy, z, size, rot, velo) 分组回归，每组独立 task head，比单一大 MLP 更稳。
关键超参汇总：num_query 900（消融）/1500（主榜）、embed_dims 256、position_range [-61.2,-61.2,-10, 61.2,61.2,10]、sweep_range [3,27]（推理 15T）、24 epoch、8×A100、batch 8、lr 2e-4 余弦退火。

10. 局限性与未来方向

PETRv2 的鲁棒性分析（§3.6 设定）量化了三类传感器误差的影响，也暴露了局限：
外参噪声（论文 Table 6，cell 为 mAP[%]）：随噪声增大三模型都掉点。极端 时 PETRv2 掉 4.12% mAP / 2.85% NDS，PETR+FPE 掉 4.68% / 3.42%，PETR 掉 6.33% / 4.54%。FPE 明显提升外参噪声鲁棒性，而多帧时序扩展本身对鲁棒性增益不大。

相机缺失：缺后相机掉 13.19% mAP（其 120° 大视场最关键），缺前相机掉 5.05%，其它相机平均掉 2.93% mAP / 1.93% NDS。提示工程上需做传感器冗余设计。
相机时延（论文 Table 7，mAP[%]，单位 ）：0.083s 时延就掉 3.19% mAP / 8.4% NDS；超过 0.3s（3T）后急剧恶化到 30.91% mAP 附近。时延在真实系统不可避免且影响显著，值得重点关注。

其它局限与未来方向：
依赖准确 ego-pose。时序对齐完全建立在相邻帧位姿（ego-motion）准确的前提上，位姿误差会直接污染对齐后的 3D 坐标。
仅用相邻两帧。论文主要用前一帧，长时序信息（多帧、长程遮挡恢复）未充分挖掘——后续 StreamPETR 正是补这一点。
全局注意力的算力。PETR 系全局 cross-attention 在高分辨率/多帧下计算量大，不像 deformable attention 那样稀疏。
无显式 BEV 的代价。少了可解释的 BEV 中间表征，对需要稠密 BEV 输出（如 occupancy）的下游任务，要额外设计 query/头。

11. 总结

PETRv2 是 PETR 的时序统一版，用三个干净的增量把「隐式 3D 位置编码」范式推到了多任务时序感知的 SOTA：
时序 3D PE 对齐（CA）：把前帧 3D 坐标经 ego-motion 对齐到当前帧，等价于对齐 3D PE——证明隐式 3D PE 能直接做时序，无需显式 BEV。速度估计大幅改善（mAVE 0.808→0.343）。
FPE：用 2D 特征（SELayer 门控）调制 3D PE 生成，从 data-independent 变 data-dependent，提升数据适应性与外参噪声鲁棒性。
多任务统一：稀疏任务专属 query（3D/BEV/anchor-lane 空间）共享 backbone+PE+decoder，一套框架做检测+分割+车道。
它在 nuScenes test 达 59.1% NDS / 50.8% mAP（多尺度），BEV 分割、OpenLane 车道线均 SOTA。工程上最值得借鉴的是「时序对齐前移到 dataloader、forward 只需统一反投影」这一极简设计。相对 BEVFormer（TSA 软注意力）和 BEVDet4D（BEV align+concat），PETRv2 选择在 隐式 3D PE 坐标层面对齐，是三种时序方案里工程代价最低的一种。
