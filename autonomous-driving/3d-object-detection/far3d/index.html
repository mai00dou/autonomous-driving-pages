1. 研究背景与动机

环视（surround-view）相机 3D 目标检测因为部署成本低、语义信息丰富，近年进展显著。但绝大多数方法只关注近距离感知（如 nuScenes 上约 50m），远距离检测（long-range）这个方向几乎没人碰。而真实驾驶里，高速行驶或复杂路况下保持安全距离，恰恰需要看得远——检测 100m、150m 外的目标。
现有环视方法按中间表征分两类：
稠密 BEV 方法（BEVDet、BEVFormer、BEVDepth 等）：用 view transformer 把透视特征转成 BEV 特征，再接 3D 检测头。问题是稠密 BEV 特征计算量大，即便近距离也很重，扩到远距离时计算量平方级膨胀，难以承受。
稀疏 query 方法（DETR3D、PETR 系、Sparse4D、StreamPETR）：沿用 DETR 范式，用可学习的全局 query 表示 3D 目标，和环视图像特征交互更新 query。稀疏设计避免了 query 数量平方增长，但固定的全局 query 无法适应动态场景，在远距离检测中经常漏检目标。
Far3D 选择稀疏 query 路线来保持效率，并引入 3D 自适应 query 来解决固定 query 不灵活的问题。
关键观察（下图）：在 Argoverse 2 上对比 StreamPETR（3D 检测）和 YOLOX（2D 检测）的召回，3D 检测召回明显偏低，而 2D 检测召回高得多。2D 检测器对远处小目标更鲁棒，可以作为「桥梁」来提升 3D 检测质量。这正是 Far3D 的出发点——用高质量 2D 先验生成 3D query。

2. 核心问题分析

要把稀疏 query 范式用到远距离检测，首要难题是召回太差。
问题一：3D 空间 query 稀疏导致召回低。 由于 3D 空间里 query 稀疏，预测与真值之间的匹配受影响，只有少量 query 能匹配成正样本。如图 1 所示，3D 检测召回很低，而 2D 检测召回高得多——两者存在显著差距。用高质量 2D 先验改进 3D proposal 是个很自然的思路。
之前 SimMOD、MV2D 也探索过用 2D 预测来初始化 3D proposal，但它们只关注近距离任务，而且丢掉了可学习的 object query。
问题二：2D 先验直接转 3D 在远距离会引入噪声。 如下图所示，直接把 2D proposal 反投影成 3D query，在远距离任务上会碰到两个麻烦：
沿目标射线的深度分布不确定，导致冗余且深度错误的预测（图 2a，黄色框就是深度错的冗余预测）；
frustum 变换的偏差随距离增大而放大——同样的 2D 误差，投影到远处会被放得很大（图 2b，误差随 range 增大被放大）。
这些噪声 query 会影响训练稳定性，需要有效的去噪手段。此外，训练过程中模型还有个倾向：过拟合密集的近处目标，而忽视稀疏分布的远处目标（range 级别的样本不均衡）。
针对以上问题，Far3D 设计了三个关键模块：自适应 query 生成、perspective-aware aggregation、range-modulated 3D denoising。

3. 技术架构详解

3.1 整体流程

Far3D 以 StreamPETR 为 baseline，整体是一个稀疏 query 框架，由四部分组成：backbone、FPN neck、2D proposal 头、3D 检测头。
整体流程（对应论文 Figure 3 Overview）：
环视图像 I = {I1, ..., In} 送入 backbone（ResNet / VoVNet / ViT）和 FPN neck，得到多级图像特征 F = {F1, ..., Fn}（默认 4 级，尺寸 1/8、1/16、1/32、1/64）。
用 2D 检测头 + 深度网络 生成可靠的 2D box proposal 和对应深度，过滤低质量的，再变换到 3D 空间，生成 3D 自适应 query。这样 2D 检测的目标先验信息就被编码进了 3D query。
在 3D 检测头里，拼接 3D 自适应 query 和 3D 全局 query，一起送入 transformer decoder。decoder 包含 query 间的 self-attention 和 query 与特征间的 perspective-aware aggregation。
同时引入 range-modulated 3D denoising，在训练长距离、分布不均衡的目标时缓解 query 误差传播、稳定收敛。
时序建模沿用 StreamPETR 的 object-centric 长序列 query 传播（memory queue 递归）。

3.2 自适应 Query 生成（Adaptive Query Generation）

这是 Far3D 的第一个核心。直接把现有 3D 检测器从近距离（~50m）扩到远距离（~150m）会遇到三个问题：计算量重、收敛慢、定位能力下降。比如要覆盖更大范围，query 数至少要平方级增长，这在实际场景里无法接受；而且远处稀疏小目标既拖慢收敛，还会伤害近处目标的定位。
Far3D 的做法：受 2D proposal 高性能启发，用 2D 检测结果作为目标先验，自适应地生成 3D query，弥补固定全局 query 的不足，让检测器能在不同图像里、在真值框附近生成自适应 query。
步骤 1：2D 检测 + 深度估计。 FPN neck 后的图像特征送入 YOLOX 的 anchor-free 检测头和一个轻量深度估计网络，输出 2D 框坐标、得分、深度图。深度估计被当成分类任务——把深度离散化成 bins（LID 离散，沿用 CaDDN / MonoDETR 思路）。然后配对 2D 框和对应深度，用得分阈值 τ（如 0.1）过滤掉低质量 proposal。
步骤 2：反投影到 3D。 对每个视图 i，2D 预测框中心 (cw, ch) 和深度图采样得到的深度 dwh 组合，投影成 3D proposal 中心 c3d：

代码落点（FarHead.build_query2d_proposal，farhead.py）。 这是自适应 query 生成的核心实现，关键是「2D 框中心采深度 → frustum lift → img2lidar 反投影 → 归一化 3D 参考点」：
@torch.no_grad()
def build_query2d_proposal(self, pred_bbox_list, pred_depth, data, bn, padHW, ...):
    B, N = bn
    pad_h, pad_w = padHW
    depth_downsample = int(pad_h / pred_depth.shape[1])   # 深度图相对原图的下采样倍数
    bbox_nums = [len(bbox) for bbox in pred_bbox_list]    # 每个视图的 2D 框数 Mi
    bboxes = torch.cat(pred_bbox_list, dim=0).float()     # (sum(Mi), 4): cw,ch,w,h
    if sum(bbox_nums) == 0:                               # 没有有效 2D proposal
        return None, None
    # —— 在深度图上按 2D 框中心位置采样对应深度 ——
    for ith, pred_bbox in enumerate(pred_bbox_list):
        cur_depthmap = pred_depth[ith].flatten(0, 1)      # (H*W, 1) 或 (H*W, D)
        cur_center2d = (pred_bbox[:, :2] / depth_downsample).round().long()  # 框中心映射到深度图网格
        # clamp 到深度图边界内...
        cur_center2d = cur_center2d.flip(dims=(-1,))      # (w,h) → (h,w)
        cur_center2d_ = cur_center2d[:, 0]*(pad_w/depth_downsample) + cur_center2d[:, 1]  # 展平索引
        cur_depth = torch.gather(cur_depthmap, 0, cur_center2d_.long().unsqueeze(1))      # (Mi, 1)
        depth_list.append(cur_depth)
    depths = torch.cat(depth_list, dim=0)                 # (M, 1) 或 (M, D)
接着是多深度候选（缓解远处深度不确定，对应 config 的 add_multi_depth_proposal + multi_depth_config={'topk':1,'range_min':30}）——一个 2D 框可生成多个 3D 候选，并把 2D 得分转成 log-odds 拼进语义 context：
    if self.add_multi_depth_proposal:
        if input_depth_logits and topk != -1:
            topk_values, topk_indices = torch.topk(depths, topk, dim=1)  # 取 topk 个深度 bin
            valid_indices = topk_indices[:, 0] >= range_min_bin          # 只对 >range_min 的远处目标加候选
            bboxes = torch.cat([bboxes, bboxes_extra[...]], dim=0)       # 框复制, 配多个深度
            depths = torch.cat([topk_indices[:, 0:1], depths_extra], 0)
        if bbox2d_scores is not None:
            thr = torch.tensor([0.1])                                    # 得分阈值 τ=0.1
            log_odds = torch.log(bbox2d_scores/(1-bbox2d_scores)) - torch.log(thr/(1-thr))
            context2d_feat = torch.cat([context2d_feat, log_odds], dim=-1)  # 2D 置信度并入语义
    depths = self._convert_bin_depth_to_specific(depths)  # LID: bin 索引 → 具体深度值(米)
最后是 frustum lift + img2lidar 反投影（对应公式 1），把图像坐标 + 深度变换到 lidar/ego 3D 空间并归一化：
    # (u,v,d) -> 齐次 (u*d, v*d, d, 1)
    coords = torch.cat([bboxes[:, :2], depths], dim=1)        # (M, 3): w,h,d
    coords = torch.cat((coords, torch.ones_like(coords[..., :1])), -1)  # (M, 4)
    coords[..., :2] = coords[..., :2] * coords[..., 2:3]      # 乘深度
    coords = coords.unsqueeze(-1)                             # (M, 4, 1)
    img2lidars = data['lidar2img'].inverse()                 # 关键: lidar2img 求逆得到反投影矩阵
    img2lidars_ = ...                                         # 按每框所属视图取对应矩阵 (M, 4, 4)
    coords3d = torch.matmul(img2lidars_, coords).squeeze(-1)[..., :3]   # (M, 3) lidar 系 3D 坐标
    # 归一化到 pc_range [-152.4,-152.4,-5, 152.4,152.4,5]
    coords3d[..., 0:1] = (coords3d[..., 0:1] - pc_range[0]) / (pc_range[3] - pc_range[0])
    coords3d[..., 1:2] = (coords3d[..., 1:2] - pc_range[1]) / (pc_range[4] - pc_range[1])
    coords3d[..., 2:3] = (coords3d[..., 2:3] - pc_range[2]) / (pc_range[5] - pc_range[2])
    new_reference_points = coords3d.unsqueeze(0)             # (B=1, M, 3) → 自适应 query 参考点
    return new_reference_points, context2d_feat
在 forward 里，自适应 query 的参考点和全局 query 的参考点直接拼接，再分别过 query embedding，组成完整的 decoder 输入：

tgt[:, -pro2d_num:, :] = self.context_embed(context_feat)         # 2D 语义注入自适应 query 的 tgt
注意 build_query2d_proposal 带 @torch.no_grad()——2D→3D 的几何反投影本身不回传梯度，自适应 query 的可学习性体现在后续 decoder 的 refine 与 embedding 上。

3.3 Perspective-aware Aggregation（透视感知聚合）

这是 Far3D 的第二个核心。现有稀疏 query 方法（如 StreamPETR）为了计算效率，通常只用单一层级的特征图。但单一特征层对不同 range 的 query 都不是最优：远处小目标需要大分辨率特征做精确定位，近处大目标更适合高层特征。
Far3D 提出 perspective-aware aggregation，在不同尺度和视图上做高效特征交互。受 deformable attention 启发，它用 3D 空间可变形注意力——由 3D 偏移采样 + 视图变换组成：
先用相机内外参（内参 I、外参 K）给图像特征 F 装配相机信息，再用一个 squeeze-and-excitation（SE）block 显式增强特征，得到增强特征 F′。
对每个 3D 空间的 query 参考点，模型学习 M 个采样偏移，把这些参考点投影到不同的 2D 尺度和视图：

其中 P3d_q、ΔP3d_q 是 query q 的 3D 参考点和学习到的偏移，P2d_q 是投影到不同尺度/视图的 2D 参考点。
3D query 根据这些 2D 参考点，与多尺度采样特征交互，按相对重要性把各视图、各尺度的特征聚合到 3D query 上。

def forward(self, instance_feature, reference_points, anchor_embed,
            feat_flatten, spatial_flatten, level_start_index, img_metas, lidar2img_mat):
    key_points = self.kps_generator(reference_points, instance_feature)   # (B, Q, 13, 3) 3D 采样点
    weights = self._get_weights(instance_feature, anchor_embed)           # 注意力权重
    features = self.feature_sampling(feat_flatten, spatial_flatten,       # 多尺度多视图采样聚合
                  level_start_index, key_points, weights, lidar2img_mat, img_metas)
    output = self.output_proj(features)
    return self.drop(output) + instance_feature                           # 残差
3D 关键点生成（SparseBox3DKeyPointsGenerator）是 perspective 采样的关键——7 个固定尺度点 + 6 个可学习点 = 13 个采样点，随框尺度 exp 缩放并按 yaw 旋转：
def forward(self, anchor, instance_feature):
    fix_scale = anchor.new_tensor(self.fix_scale)        # 7 个固定点: 中心 + 6 个 ±0.45 轴向点
    scale = fix_scale[None, None].repeat(bs, num_anchor, 1, 1)
    if self.num_learnable_points > 0:                    # 6 个可学习偏移点
        learnable_scale = self.learnable_fc(instance_feature).reshape(
            bs, num_anchor, self.num_learnable_points, 3).sigmoid() - 0.5
        scale = torch.cat([scale, learnable_scale], dim=2)   # 共 13 点
    key_points = scale * anchor[..., None, 3:6].exp()    # 按框尺寸 (w,l,h) 缩放
    # 按 yaw (anchor[...,6:7]=sin, [...,7:8]=cos) 构造旋转矩阵并旋转
    key_points = torch.matmul(rotation_mat[:, :, None], key_points[..., None]).squeeze(-1)
    key_points = key_points + anchor[..., None, :3]      # 平移到参考点中心
    return key_points
特征采样（feature_sampling）：3D 关键点用 lidar2img 投影到各视图各尺度，归一化后调用 CUDA 的 MultiScaleDeformableAttn 核做高效采样，再按 group 加权融合：
def feature_sampling(self, feat_flatten, spatial_flatten, level_start_index,
                     key_points, weights, lidar2img_mat, img_metas):
    pts_extand = torch.cat([key_points, torch.ones_like(key_points[..., :1])], dim=-1)
    points_2d = torch.matmul(lidar2img_mat[:, :, None, None], pts_extand[:, None, ..., None]).squeeze(-1)
    points_2d = points_2d[..., :2] / torch.clamp(points_2d[..., 2:3], min=1e-5)  # 透视除法
    points_2d[..., 0:1] /= img_metas[0]['pad_shape'][0][1]   # 归一化到图像宽
    points_2d[..., 1:2] /= img_metas[0]['pad_shape'][0][0]   # 归一化到图像高
    points_2d = points_2d.flatten(end_dim=1)[:, :, None, None, :, :].repeat(
        1, 1, self.num_groups, self.num_levels, 1, 1)        # 扩展到 groups × levels
    output = MultiScaleDeformableAttnFunction.apply(         # CUDA 多尺度可变形采样
        feat_flatten, spatial_flatten, level_start_index, points_2d, weights, self.im2col_step)
    output = output.reshape(bs, self.num_cams, num_anchor, -1)
    return output.sum(1)                                     # 跨 7 视图求和
权重 _get_weights 对 cams×levels×pts 维度做 softmax——这正是「按透视/距离自适应选尺度选视图」的落点：远处小目标的 query 会把权重集中到大分辨率低层特征，近处大目标偏向高层特征，全部由网络学习。这套 deformable aggregation 框架直接沿用自 Sparse4D 系（稀疏 query + 3D 关键点可变形采样），是 Far3D 与 StreamPETR（全局注意力 + 单层特征）最大的工程区别。

3.4 时序建模与三类 Query

Far3D 沿用 StreamPETR 的 object-centric 时序机制：用上一帧的高分 query 作为传播 query（propagated query），递归地把时序特征带到当前帧（memory queue，config 中 memory_len=1024、num_propagated=256）。
所以 Far3D 里一共有三类 query（论文 Figure 6）：
全局 query（global query）：从数据学到的固定 query，config 中 num_query=644。
自适应 query（adaptive query）：由当前帧 2D proposal 生成，数量动态（AV2 平均 92 个/帧）。
传播 query（propagated query）：来自前一帧的高分 query，承载时序信息——选择只看 query 得分，与 query 原本属于哪类无关。

5. 详细流程图：Range-modulated 3D Denoising（范围调制去噪）

这是 Far3D 的第三个核心。不同距离的 3D query 回归难度不同——这点和 2D 去噪（如 DN-DETR 对所有 query 一视同仁）很不一样。难度差异来自两方面：
query 密度：远处目标对应的 query 比近处少、更难匹配；
误差传播：2D proposal 的小误差，引入到 3D 自适应 query 时会被放大（图 2），而且距离越远效果越严重。
所以 GT 框附近的某些 query proposal 可以当作噪声正候选，另一些偏差大的应当当负样本。Far3D 的目标：召回潜在正样本、直接拒绝确定的负样本——这就是 range-modulated 3D denoising。
做法：基于 GT 同时构造正、负两组噪声 query（公式 6），按目标位置和尺寸加随机噪声。正样本噪声受框尺度约束在框内；负样本噪声与位置 range 相关（  形式最优）。每个 GT 多组样本，每组 1 正 + K 负。这样既缓解了 2D 先验误差传播，又缓解了 range 级别的样本不均衡。
代码落点（FarHead.prepare_for_dn，farhead.py）。 核心是正/负样本用不同噪声策略，且负样本噪声按坐标绝对值取对数（range-modulated 的字面落点）：
# 正样本噪声: 与框尺度成线性, 约束在框内 (offset_p, bbox_noise_trans)
diff_p = known_bbox_scale / 2 + self.bbox_noise_trans
diff_p = torch.mul(torch.rand_like(known_bbox_center) + self.offset_p, diff_p) * self.bbox_noise_scale
rand_sign = torch.randint_like(known_bbox_center, 0, 2) * 2.0 - 1.0
known_bbox_center_p = known_bbox_center + torch.mul(rand_sign, diff_p)   # 正样本中心
# 负样本噪声: range-modulated —— 按中心坐标绝对值取 log, 距离越远噪声越大
diff_n = (known_bbox_center_ori.abs() + 1).log()                        # <<< log 形式 (论文最优)
diff_n = torch.mul(torch.rand_like(...) + self.offset, diff_n)
rand_sign_n = torch.randint_like(...) * 2.0 - 1.0
known_bbox_center_n = known_bbox_center_ori + torch.mul(rand_sign_n, diff_n)  # 负样本中心
config 对应超参：scalar=10（噪声组数）、num_smp_per_gt=3（每 GT 1 正 2 负）、split=0.75（正样本率）、offset=0.5、noise_scale=1.0、dn_weight=1.0。去噪 query 在 decoder 里通过 attn_mask 与正常匹配 query 隔离（匹配 query 看不到去噪组，去噪组之间也互不可见），与 DN-DETR 一致。
prepare_for_dn 还用 torch.cdist + linear_sum_assignment 给每组噪声样本做匹配（因为一个 GT 对应多个候选，需要匈牙利分配确定哪个是该恢复的正样本）。这是相比标准 DN-DETR（噪声样本与 GT 一一对应、无需匹配）的改动。

6. 核心创新点

Far3D 强调它的主要贡献是面向远距离 3D 检测的统一框架，而不是孤立的组件。三个创新点协同解决远距离的「召回差、计算重、误差传播」三大痛点：
创新 1：用 2D 先验生成 3D 自适应 query（核心）。 这是 Far3D 与所有纯固定 query 方法（StreamPETR、Sparse4D）最本质的区别。2D 检测器召回高、对远处小目标鲁棒，把 2D box + 深度反投影成 3D query，相当于给固定全局 query「补充」了一批落在真值附近、随场景动态变化的 query。论文统计：AV2 上自适应 query 平均每帧 92 个（最多 236，最少 11），只占总 query 的小部分，但贡献显著——把这 92 个换成额外的固定 query，mAP 掉 1.5%、CDS 掉 1.3%。
创新 2：perspective-aware aggregation。 用 3D 可变形采样在多尺度（p2–p5）多视图上自适应聚合特征。远处小目标偏向大分辨率低层特征，近处大目标偏向高层特征，权重全由网络学习。相比 StreamPETR 的单层全局注意力，既省算力又提精度。
创新 3：range-modulated 3D denoising。 针对远距离误差传播和样本不均衡，正样本约束在框内、负样本噪声按距离 log 调制，召回潜在正候选、拒绝假正。它同时缓解了 range 级别的不均衡分布问题。

7. 实验结果与分析

7.1 数据集与指标

Argoverse 2（AV2）：长距离自动驾驶数据集，1000 个场景，每场景 15 秒、10Hz 标注，700/150/150 划分 train/val/test。7 路高分辨率 ring 相机，360° 视野。评测 26 个类别、150m 范围——这正是远距离任务。指标：mAP、CDS（Composite Detection Score，AV2 主指标）、ATE/ASE/AOE。
nuScenes：1000 个驾驶场景，10 类，~50m 近距离。用来验证泛化能力。指标：mAP、NDS。
实现细节：以 StreamPETR 为 baseline，VoVNet-99（FCOS3D 预训练）做主实验，ViT-L（Objects365 + COCO 预训练）做 scale-up。FPN 给 4 级特征（1/8、1/16、1/32、1/64）。感知范围 152.4m × 152.4m。AdamW，weight decay 0.01，total batch size 8，lr 2e-4，训 6 epoch。

7.2 主榜结果

Argoverse 2（下表 Table 1）：VoV-99 + 960×640 输入下，Far3D 达到 24.4 mAP / 18.1 CDS，比 StreamPETR（20.3 / 14.6）高出 4.1% mAP、3.5% CDS，大幅超越其它环视方法。换 ViT-L + 1536×1536，Far3D 达 31.6 mAP / 23.9 CDS，甚至超过了若干 LiDAR 方法（CenterPoint 27.4、FSD 29.1、VoxelNeXt 30.7 mAP）。细节上 LiDAR 方法定位误差（ATE）更低（深度准），而环视方法朝向估计（AOE）更好。

8. 技术发展时间线

Far3D 站在 PETR/Sparse4D 谱系肩膀上，整合了多条技术线：
DETR (2020)：query + 二分匹配的端到端检测范式。
Deformable DETR (2020)：可变形注意力，稀疏采样降复杂度——Far3D 的 perspective-aware aggregation 直接继承。
DETR3D (2021)：3D query 投影回 2D 采样特征，开启稀疏 query 3D 检测。
PETR / PETRv2 (2022)：3D 位置编码 + 时序对齐。
DN-DETR (2022)：query 去噪加速收敛——Far3D 的 3D denoising 的来源，但改成 range-modulated。
CaDDN / MonoDETR：深度分类（bin 离散化）——Far3D 的 DepthNet 思路。
StreamPETR (2023)：object-centric 长时序 query 传播——Far3D 的 baseline 与时序机制。
Sparse4D / Sparse4D v2 (2023)：稀疏 query + 3D 关键点可变形聚合 + 递归时序——Far3D 的 deformable aggregation 框架来源。
SimMOD / MV2D / BEVFormerV2：用 2D proposal/监督辅助 3D——Far3D 的自适应 query 思想前身，但 Far3D 保留了可学习全局 query 并专攻远距离。
Far3D (AAAI 2024)：首个把稀疏 query 3D 检测系统性扩展到 150m 远距离的框架。

9.4 几个实现要点

自适应 query 生成不回梯度：build_query2d_proposal 带 @torch.no_grad()，纯几何反投影。可学习性在 decoder refine 和各 embedding。
早期 GT 深度热身：训练前期用 GT 深度生成自适应 query 保证 query 质量，待网络稳定后切预测深度（UseGtDepthHook）。这缓解了「2D 误差传播」对早期训练的冲击。
2D 得分注入语义：2D 框置信度转成 log-odds（log(s/(1-s)) - log(τ/(1-τ))）拼进 query 的语义特征，让高分 proposal 的 query 权重更大。
去噪 query 需匈牙利匹配：因一个 GT 配多个候选样本，用 linear_sum_assignment 在 cost 矩阵上分配，区别于标准 DN-DETR 的一一对应。
B=1 限制：build_query2d_proposal 里 if B == 1 才支持，else: raise NotImplementedError——官方实现只测了 batch size 1（config 里 batch_size=1）。

10. 局限性与未来方向

作者指出两点局限：
长尾类别表现差：现有方法在长尾类别上性能仍弱，拉低了 AV2 的平均精度（AV2 有 26 类，远多于其它数据集）。
统一指标不合理：用同一套指标评测远距离和近距离目标可能不合适，需要更实用、动态、面向多样真实场景的评测标准。
此外补充材料里给了一个有意思的对比（下图 Figure 7）：Far3D vs SOLOFusion（典型 BEV 方法）的可视化。SOLOFusion 即使用了 NMS 仍产生大量重复预测，作者推测这源于检测头在大感知范围下感受野受限。相比之下 Far3D 的稀疏 query 设计在大范围下更干净。

11. 总结

Far3D 是首个把**稀疏 query 3D 检测系统性扩展到远距离（150m）**的框架。核心思路：纯固定 query 方法（StreamPETR、Sparse4D）在远处召回低，于是用 2D 检测的高召回先验生成 3D 自适应 query 来补充固定全局 query；再用 perspective-aware aggregation 在多尺度多视图上自适应聚合特征，缓解远处特征采样误差；最后用 range-modulated 3D denoising 应对 2D→3D 的误差传播和样本不均衡、稳定收敛。
成绩：Argoverse 2（150m、26 类）上大幅超越环视方法，ViT-L 下甚至超过若干 LiDAR 方法；nuScenes 上验证了泛化性（63.5 mAP / 68.7 NDS test）。
与 StreamPETR / Sparse4D 的本质区别：
StreamPETR/Sparse4D = 固定数量全局 query + 时序传播，近距离强但远处召回不足；
Far3D = 固定全局 query + 动态自适应 query（2D 先验生成） + perspective 多尺度可变形聚合 + 3D 去噪，专门补齐远距离/小目标召回。
工程框架上，Far3D 的可变形聚合沿用 Sparse4D 系（3D 关键点采样），时序沿用 StreamPETR（object-centric memory queue），是两条线的融合 + 2D 先验创新。
