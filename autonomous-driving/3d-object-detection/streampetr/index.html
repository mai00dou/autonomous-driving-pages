1. 研究背景与动机

纯相机的多视角 3D 目标检测在自动驾驶里非常关键：相机部署成本低、对车道线红绿灯等路面元素友好。近两年大家普遍发现，引入时序信息能显著缓解遮挡、并降低速度估计的难度，因此时序建模成为多视角 3D 检测的主战场。按照「时序表征用什么载体」来分，已有方法大致分两类：
BEV 时序方法（BEVFormer、BEVDet4D、BEVDepth、SOLOFusion 等）：把历史帧的 BEV 特征显式 warp 到当前帧再融合。BEV 特征是一个高效的中间表征，但它是高度结构化的稠密栅格，对运动物体的建模能力受限——为了缓解这个问题往往需要很大的感受野，或者把很多历史帧叠在一起。
透视（perspective）时序方法（PETRv2、Sparse4D、DETR4D 等）：基于 DETR 的稀疏 query 设计，query 直接和多帧透视图像特征做 cross-attention。稀疏 query 天然适合建模运动物体，但每加一帧历史就要重新和那一帧的图像特征交互一次，长时序意味着重复多次的特征聚合计算，开销随帧数线性增长。
两条路各有短板：要么卡在运动物体建模，要么卡在长时序的算力开销。
作者的核心洞察是：能不能把稀疏 object query 直接当作时序传播的「隐状态」？ 这样既能用 query 的稀疏性来建模运动物体，又能像 RNN 那样把长时序信息低成本地递归累积下来。基于这个想法，他们提出了一个新范式——object-centric 时序建模，并设计了高效框架 StreamPETR，在流式视频上逐帧做 3D 预测。
具体而言，StreamPETR 先建一个 memory queue 存历史 object query，再用一个 propagation transformer 让当前 query 与历史 query、当前图像特征做长程时空交互；更新后的 query 既用来出 3D 框，又被推回 memory queue；此外引入 motion-aware layer normalization（MLN），隐式编码自车（ego）与周围物体在不同时间戳下的运动。
相比已有时序方法，object-centric 范式带来几个好处：每帧只处理少量 object query 而非稠密特征图，计算负担可忽略；MLN 缓解了视频流里运动物体的累积误差；除了以往方法用的位置先验外，StreamPETR 还通过全局注意力额外考虑了语义相似性，利于运动场景下的检测。下面这张速度-精度权衡图直观地展示了它的优势：在 nuScenes val 上，轻量版达到 45.0% mAP、31.7 FPS，比当时 SOTA 的 SOLOFusion 高 2.3% mAP 且速度快 1.8 倍。

提出 StreamPETR 框架，同时建模运动物体和长程时空交互，存储与计算开销都可忽略。

2. 核心问题分析

2.1 多视角 3D 检测里的时序难题

多视角 3D 检测要持续处理多相机图像并随时间预测 3D 框。早期工作聚焦于「单帧内」如何把多个透视视图高效变换到统一 3D 空间，分为 BEV 类方法和稀疏 query 类方法。为了缓解遮挡、降低速度预测难度，近期工作给这两条路都加上了时序。
把单帧 BEV 方法扩展到时序相对直观：BEVFormer 最早把序列时序建模引入多视角 3D 检测，用 temporal self-attention；BEVDet 系列用 concat 操作融合相邻 BEV 特征；SOLOFusion 把 BEVStereo 扩展到长时序记忆，性能很强。但这些方法的时序融合只考虑静态 BEV 特征，忽略了物体运动，会带来空间错位（spatial dislocation）。
没有中间特征表征的 query 类方法，时序建模更难：PETRv2 做全局 cross-attention，DETR4D 和 Sparse4D 用稀疏注意力建模多帧交互——但都引入了多次重复计算。好处是稀疏 query 设计方便建模运动物体。作者想结合两者优点：用稀疏 object query 当中间表征，既能建模运动物体，又能高效传播长时序信息。

2.2 query propagation 的思想来源

「用 query 传播时序」并非凭空而来。自 DETR 在 2D 检测里提出 object query，它已被用于很多下游任务建模时序交互：
视频目标检测：LWDN 用类脑记忆机制传播更新记忆特征；QueryProp 在非关键帧做 query 交互降算力；3D-MAN 把单帧 LiDAR 检测器扩展到多帧。
目标跟踪：MOTR、TrackFormer 提出 track query 建模跨帧关联；MeMOT 用 memory bank 建长时序依赖；MOTRv2 引入额外检测器缓解检测与关联的冲突；MUTR、PF-Track 把 MOTR 扩展到多视角 3D 跟踪。
StreamPETR 把这套「query 作为时序载体」的思想，系统化地落到多视角 3D 检测上，并配以专门的运动补偿（MLN）和高效的 hybrid attention。

3. 技术架构详解

3.1 整体架构

StreamPETR 建立在端到端稀疏 query 的 3D 检测器（PETR / DETR3D）之上，由三部分组成：图像编码器、递归更新的 memory queue、传播 transformer（propagation transformer）。图像编码器是标准 2D backbone（ResNet / V2-99 / ViT），从多视角图像抽语义特征；抽出的特征、memory queue 里的信息、object query 一起喂进传播 transformer 做时空交互。
StreamPETR 与单帧 baseline 最大的区别就是这个 memory queue——它递归更新 object query 的时序信息，配合传播 transformer，把时序先验从过去帧高效传到当前帧。下面是论文 Fig. 3 的整体架构图：

memory queue 设计为  ：  是存的帧数，  是每帧存的物体数。论文经验取  、 （保证复杂场景下高召回）。每隔预设时间间隔  ，把选中 query 的相对时间  、上下文嵌入  、物体中心  、速度  、ego-pose 矩阵   存入队列。具体只存前景物体（分类分数 top-K）的上述信息。出入队走 FIFO：新帧信息进来，最老的丢掉。队列高度可定制——用户可在训练和推理时自由控制最大记忆   和保存间隔  。
工程实现关键：仓库里 memory queue 不是一个真的 Python 队列，而是几个常驻在 head 上的张量缓存（memory_embedding / memory_reference_point / memory_timestamp / memory_egopose / memory_velo），靠「在 dim=1 上 cat 新帧、再截断到 memory_len」来模拟 FIFO。
memory 的初始化与重置（StreamPETRHead.reset_memory，文件 projects/mmdet3d_plugin/models/dense_heads/streampetr_head.py）：
def reset_memory(self):
    # 切换场景 / 新序列时把 5 路记忆全部清空
    self.memory_embedding = None        # 历史 query 的 context 嵌入 Q_c
    self.memory_reference_point = None  # 历史 query 的 3D 中心 Q_p
    self.memory_timestamp = None        # 历史 query 的时间戳（用于算 Δt）
    self.memory_egopose = None          # 历史 query 对应的 ego-pose 矩阵 E
    self.memory_velo = None             # 历史 query 的速度 v
每帧推理开头：pre_update_memory（出队 + 把历史坐标对齐到当前帧）。这是 object-centric 传播的「读」阶段：
def pre_update_memory(self, data):
    x = data['prev_exists']   # 标志位: 0=新序列首帧, 1=延续上一帧
    B = x.size(0)
    if self.memory_embedding is None:
        # 首帧: 5 路记忆 zero 初始化, 形状 (B, memory_len, ...)
        self.memory_embedding       = x.new_zeros(B, self.memory_len, self.embed_dims)
        self.memory_reference_point = x.new_zeros(B, self.memory_len, 3)
        self.memory_timestamp       = x.new_zeros(B, self.memory_len, 1)
        self.memory_egopose         = x.new_zeros(B, self.memory_len, 4, 4)
        self.memory_velo            = x.new_zeros(B, self.memory_len, 2)
    else:
        # 1) 时间戳累加当前帧时间间隔 → 得到历史 query 相对当前帧的 Δt
        self.memory_timestamp += data['timestamp'].unsqueeze(-1).unsqueeze(-1)
        # 2) 把历史 ego-pose 左乘当前帧 ego_pose_inv → 转换到当前帧坐标系
        self.memory_egopose = data['ego_pose_inv'].unsqueeze(1) @ self.memory_egopose
        # 3) 历史 3D 中心也用 ego_pose_inv 对齐到当前帧 (运动补偿的"显式"部分)
        self.memory_reference_point = transform_reference_points(
            self.memory_reference_point, data['ego_pose_inv'], reverse=False)
        # 4) 截断到 memory_len, 并用 prev_exists 做 mask: 新序列首帧把记忆清零
        self.memory_timestamp       = memory_refresh(self.memory_timestamp[:, :self.memory_len], x)
        self.memory_reference_point = memory_refresh(self.memory_reference_point[:, :self.memory_len], x)
        self.memory_embedding       = memory_refresh(self.memory_embedding[:, :self.memory_len], x)
        self.memory_egopose         = memory_refresh(self.memory_egopose[:, :self.memory_len], x)
        self.memory_velo            = memory_refresh(self.memory_velo[:, :self.memory_len], x)
    # 首帧没有历史 query, 用非学习的 pseudo reference points 占位 num_propagated 个
    if self.num_propagated > 0:
        pseudo_reference_points = self.pseudo_reference_points.weight \
            * (self.pc_range[3:6] - self.pc_range[0:3]) + self.pc_range[0:3]
        self.memory_reference_point[:, :self.num_propagated] = \
            self.memory_reference_point[:, :self.num_propagated] \
            + (1 - x).view(B, 1, 1) * pseudo_reference_points
        self.memory_egopose[:, :self.num_propagated] = \
            self.memory_egopose[:, :self.num_propagated] \
            + (1 - x).view(B, 1, 1, 1) * torch.eye(4, device=x.device)
这里 memory_refresh(memory, prev_exist) 极简：return memory * prev_exist——用 0/1 标志位把新序列首帧的记忆乘成 0，等价于「换场景就清空记忆」。
每帧推理结尾：post_update_memory（入队 + Top-K 筛选）。这是 object-centric 传播的「写」阶段：
def post_update_memory(self, data, rec_ego_pose, all_cls_scores, all_bbox_preds, outs_dec, mask_dict):
    # 取最后一层 decoder 的输出作为本帧 query 状态(训练时跳过 dn 的 pad 部分)
    rec_reference_points = all_bbox_preds[..., :3][-1]   # 本帧预测的 3D 中心
    rec_velo   = all_bbox_preds[..., -2:][-1]            # 速度
    rec_memory = outs_dec[-1]                            # context 嵌入
    rec_score  = all_cls_scores[-1].sigmoid().topk(1, dim=-1).values[..., 0:1]  # 前景分数
    rec_timestamp = torch.zeros_like(rec_score, dtype=torch.float64)            # 当前帧 Δt=0
    # ① Top-K 前景筛选: 按分类分数选 topk_proposals 个 query 入队
    _, topk_indexes = torch.topk(rec_score, self.topk_proposals, dim=1)
    rec_timestamp        = topk_gather(rec_timestamp, topk_indexes)
    rec_reference_points = topk_gather(rec_reference_points, topk_indexes).detach()
    rec_memory           = topk_gather(rec_memory, topk_indexes).detach()  # detach: 不让梯度跨帧
    rec_ego_pose         = topk_gather(rec_ego_pose, topk_indexes)
    rec_velo             = topk_gather(rec_velo, topk_indexes).detach()
    # ② 入队: 新帧 query 拼到队列最前面 (FIFO 的"进")
    self.memory_embedding       = torch.cat([rec_memory, self.memory_embedding], dim=1)
    self.memory_timestamp       = torch.cat([rec_timestamp, self.memory_timestamp], dim=1)
    self.memory_egopose         = torch.cat([rec_ego_pose, self.memory_egopose], dim=1)
    self.memory_reference_point = torch.cat([rec_reference_points, self.memory_reference_point], dim=1)
    self.memory_velo            = torch.cat([rec_velo, self.memory_velo], dim=1)
    # ③ 把刚入队的中心/时间/ego-pose 转回"当前帧的全局参照", 供下一帧对齐
    self.memory_reference_point = transform_reference_points(
        self.memory_reference_point, data['ego_pose'], reverse=False)
    self.memory_timestamp -= data['timestamp'].unsqueeze(-1).unsqueeze(-1)
    self.memory_egopose = data['ego_pose'].unsqueeze(1) @ self.memory_egopose
几个值得敲黑板的设计：
Top-K 入队（topk_proposals，配置里 R50 取 128）：只把分类分数最高的前景 query 推进队列，背景 query 直接丢弃，既省存储又保证队列里都是「有用」的物体隐状态。
.detach() 跨帧切断梯度：入队的 query 状态全部 detach，历史 query 在当前帧只作「只读先验」，梯度不会反传到过去帧——这是流式训练能稳定、且开销低的关键。
坐标系来回搬运：pre 阶段用 ego_pose_inv 把历史搬到「当前帧系」做对齐，post 阶段用 ego_pose 搬回「全局系」存着，下一帧再搬。这套 ego 变换就是运动补偿里「显式对齐自车运动」的部分。

3.3 Propagation Transformer（传播 transformer）

如论文 Fig. 4 所示，传播 transformer 由三大组件构成：(1) motion-aware layer normalization（MLN），按 memory queue 里记录的上下文嵌入和运动信息隐式更新物体状态；(2) hybrid attention，替代默认的 self-attention，承担时序建模 + 去除重复预测的双重角色；(3) cross-attention，做特征聚合，可换成任意空间操作来建立图像 token 与 3D object query 的关系（PETR 的全局注意力或 DETR3D 的稀疏投影注意力）。

MLN 的代码实现（projects/mmdet3d_plugin/models/utils/misc.py）正是一个标准的条件 LN：
class MLN(nn.Module):
    '''条件层归一化: 用运动属性 c 调制特征 x 的 LN 仿射参数
       c_dim: 运动条件向量维度;  f_dim: 特征维度(256)'''
    def __init__(self, c_dim, f_dim=256):
        super().__init__()
        self.reduce = nn.Sequential(nn.Linear(c_dim, f_dim), nn.ReLU())
        self.gamma  = nn.Linear(f_dim, f_dim)   # 生成仿射 scale γ
        self.beta   = nn.Linear(f_dim, f_dim)   # 生成仿射 shift β
        self.ln     = nn.LayerNorm(f_dim, elementwise_affine=False)  # 关掉默认仿射!
        self.reset_parameters()
    def reset_parameters(self):
        # 关键初始化: γ→1, β→0, 让 MLN 初始等价于普通 LN, 训练再慢慢学运动调制
        nn.init.zeros_(self.gamma.weight); nn.init.ones_(self.gamma.bias)
        nn.init.zeros_(self.beta.weight);  nn.init.zeros_(self.beta.bias)
    def forward(self, x, c):
        x = self.ln(x)              # 先无仿射 LN
        c = self.reduce(c)          # 运动条件降到 f_dim
        gamma = self.gamma(c)       # γ = ξ1(M)
        beta  = self.beta(c)        # β = ξ2(M)
        out = gamma * x + beta      # 运动条件仿射: 式(11)
        return out
elementwise_affine=False + 外部生成   就是「conditional LayerNorm」的标准写法；reset_parameters 把   初始化成 1、  成 0，保证训练初期 MLN ≈ 普通 LN，避免运动调制一上来就扰乱训练。

def temporal_alignment(self, query_pos, tgt, reference_points):
    B = query_pos.size(0)
    # 历史中心归一化 + 生成历史 query 的位置编码 temp_pos
    temp_reference_point = (self.memory_reference_point - self.pc_range[:3]) \
                           / (self.pc_range[3:6] - self.pc_range[0:3])
    temp_pos = self.query_embedding(pos2posemb3d(temp_reference_point))
    temp_memory = self.memory_embedding
    rec_ego_pose = torch.eye(4, ...).repeat(B, query_pos.size(1), 1, 1)
    if self.with_ego_pos:
        # 当前 query: 运动属性为零(当前帧 Δt=0,v=0), 仅 ego 单位阵 → nerf 编码 → MLN
        rec_ego_motion = torch.cat([torch.zeros_like(reference_points[...,:3]),
                                    rec_ego_pose[..., :3, :].flatten(-2)], dim=-1)
        rec_ego_motion = nerf_positional_encoding(rec_ego_motion)
        tgt       = self.ego_pose_memory(tgt, rec_ego_motion)        # 调制当前 context
        query_pos = self.ego_pose_pe(query_pos, rec_ego_motion)      # 调制当前位置编码
        # 历史 query: 用真实的 速度+时间戳+ego-pose 拼成运动条件 M → nerf 编码 → MLN
        memory_ego_motion = torch.cat([self.memory_velo, self.memory_timestamp,
                                       self.memory_egopose[..., :3, :].flatten(-2)], dim=-1).float()
        memory_ego_motion = nerf_positional_encoding(memory_ego_motion)
        temp_pos    = self.ego_pose_pe(temp_pos, memory_ego_motion)     # 运动补偿历史位置编码
        temp_memory = self.ego_pose_memory(temp_memory, memory_ego_motion)  # 运动补偿历史 context
    # 时间嵌入: 当前帧 Δt=0, 历史帧用真实 Δt
    query_pos += self.time_embedding(pos2posemb1d(torch.zeros_like(reference_points[...,:1])))
    temp_pos  += self.time_embedding(pos2posemb1d(self.memory_timestamp).float())
    # ★ 传播 query: 把 memory 里前 num_propagated 个历史 query 直接拼进当前帧 query 集
    if self.num_propagated > 0:
        tgt              = torch.cat([tgt, temp_memory[:, :self.num_propagated]], dim=1)
        query_pos        = torch.cat([query_pos, temp_pos[:, :self.num_propagated]], dim=1)
        reference_points = torch.cat([reference_points,
                                      temp_reference_point[:, :self.num_propagated]], dim=1)
        temp_memory = temp_memory[:, self.num_propagated:]   # 其余留给 hybrid attention 当 K/V
        temp_pos    = temp_pos[:, self.num_propagated:]
    return tgt, query_pos, reference_points, temp_memory, temp_pos, rec_ego_pose
这段是 object-centric 传播的「机枢」，两件事尤其要看清：
运动补偿对历史和当前 query 区别对待：当前帧 query 的运动条件是「零速度、零时间、单位 ego 阵」，等于不补偿；历史 query 则用真实的 memory_velo / memory_timestamp / memory_egopose 拼成运动条件  ，经 MLN 把它们「推」到当前时刻应有的位置和状态。这正是论文式 (6)(7) 里   的落地。
传播 query（num_propagated）与 hybrid memory 的分工：memory queue 里前 num_propagated 个历史 query 被直接拼进当前帧的 query 集合（变成「带着历史身份」的检测 query，配置里 R50 取 128）；剩下的历史 query 作为 hybrid attention 的 key/value（temp_memory），只参与注意力交互不直接出框。前者让物体「跨帧续命」，后者提供更长的时序上下文。
为什么用隐式 MLN 而非显式速度外推？论文消融（Tab. 6）显示显式运动补偿（MC）反而不涨点——作者认为显式方式在训练早期容易引发误差传播；MLN 隐式编码 + 解耦自车与物体运动更稳，单是隐式编码 ego-pose 就带来 mAP +2.0%、NDS +1.8%。

3.5 Hybrid Attention（混合注意力）——代码级拆解

DETR 里的 self-attention 负责去除重复预测。StreamPETR 把它替换成 hybrid attention，额外引入时序交互：把 memory queue 里所有历史 query 和当前 query 拼起来，得到 hybrid query；hybrid query 作为多头注意力的 key 和 value。由于 hybrid query 数量很小（约 2k，远小于 cross-attention 里的图像 token），hybrid attention 的计算开销可忽略。
实现落在传播 transformer 的 decoder layer _forward 里（projects/mmdet3d_plugin/models/utils/petr_transformer.py，PETRTemporalDecoderLayer）。注意它复用了 mmcv 的 operation_order 机制，但在 self_attn 分支里做了「拼 temp_memory」的关键改动：
for layer in self.operation_order:    # ('self_attn','norm','cross_attn','norm','ffn','norm')
    if layer == 'self_attn':
        if temp_memory is not None:
            # ★ Hybrid Attention 的本质: Key/Value = [当前 query, 历史 memory query]
            temp_key = temp_value = torch.cat([query, temp_memory], dim=0)
            temp_pos = torch.cat([query_pos, temp_pos], dim=0)
        else:
            temp_key = temp_value = query   # 退化成普通 self-attn
            temp_pos = query_pos
        query = self.attentions[attn_index](
            query,            # Query = 当前帧 query (含传播 query)
            temp_key,         # Key   = 当前 query ⊕ 历史 memory query
            temp_value,       # Value = 同上
            identity if self.pre_norm else None,
            query_pos=query_pos,
            key_pos=temp_pos,
            attn_mask=attn_masks[attn_index],   # 训练时叠加 dn 的 temporal_attn_mask
            key_padding_mask=query_key_padding_mask,
            **kwargs)
        attn_index += 1; identity = query
    elif layer == 'cross_attn':
        # 当前 query ↔ 图像 token (key=memory 图像特征, key_pos=3D PE)
        query = self.attentions[attn_index](
            query, key, value, identity if self.pre_norm else None,
            query_pos=query_pos, key_pos=key_pos,
            attn_mask=attn_masks[attn_index], key_padding_mask=key_padding_mask, **kwargs)
        attn_index += 1; identity = query
    elif layer == 'ffn':
        query = self.ffns[ffn_index](query, identity if self.pre_norm else None)
配置里两个 attention 的类型是有讲究的：self_attn（即 hybrid attention）用普通 MultiheadAttention（query 数少，无需优化）；cross_attn（图像特征聚合，token 多）用 PETRMultiheadFlashAttention 省显存。decoder 堆 6 层（num_layers=6），每层都做一次 hybrid + cross，对应附录 Alg. 1 里 for i ∈ L 的层层精炼。

3.6 PETR 风格的 3D 位置编码

StreamPETR 沿用 PETR 的 3D PE 思路：在每个相机视锥里采样一串深度点，反投影到 3D 空间，再做正弦位置编码，让 2D 图像 token 带上 3D 位置先验。核心在 head 的 position_embeding：
def position_embeding(self, data, memory_centers, topk_indexes, img_metas):
    # 沿深度方向取 D 个采样点 coords_d (LID 非线性离散化或线性)
    coords = torch.cat([topk_centers, coords_d], dim=-1)  # 像素(u,v) + 深度 d
    coords = torch.cat((coords, torch.ones_like(coords[..., :1])), -1)  # 齐次
    coords[..., :2] = coords[..., :2] * torch.maximum(coords[..., 2:3], eps)  # 反投影前乘深度
    # 用 img2lidar (lidar2img 的逆) 把视锥点变到 3D lidar 系
    img2lidars = data['lidar2img'].inverse()
    coords3d = torch.matmul(img2lidars, coords).squeeze(-1)[..., :3]
    coords3d = (coords3d - self.position_range[0:3]) / (self.position_range[3:6] - self.position_range[0:3])
    pos_embed = inverse_sigmoid(coords3d)
    coords_position_embeding = self.position_encoder(pos_embed)  # MLP → 256 维 3D PE
    # cone: 给 Focal-PETR 的相机参数空间对齐(含相机内参 + 远近平面 3D 点)
    cone = torch.cat([intrinsic, coords3d[..., -3:], coords3d[..., -90:-87]], dim=-1)
    return coords_position_embeding, cone
与 PETR 不同的是，StreamPETR 还把 cone（相机内参 + 视锥远近端点）喂给 spatial_alignment = MLN(8) 做 Focal-PETR 式的相机参数对齐，再用 featurized_pe（一个 SE 风格的 Linear gate）把 3D PE 和图像特征做特征化融合。这部分继承自 Focal-PETR，本文实现也确实基于 Focal-PETR（带 2D 辅助监督）。

    memory = self.memory_embed(memory)
    memory = self.spatial_alignment(memory, cone)        # MLN(8) 相机参数对齐
    pos_embed = self.featurized_pe(pos_embed, memory)
    reference_points = self.reference_points.weight       # 300 个可学习 anchor

        self.temporal_alignment(query_pos, tgt, reference_points)

8. 技术发展时间线（PETR 谱系视角）

把 StreamPETR 放到多视角 3D 检测时序建模的演进脉络里看：
DETR (2020)：提出 object query + 集合预测，奠定稀疏 query 检测范式。
Deformable DETR (2020)：稀疏可变形注意力，加速收敛、降算力。
DETR3D (2021)：3D-to-2D query 投影，把 DETR 搬到多视角 3D，单帧。
PETR (2022)：3D 位置编码（隐式），免去显式投影，全局 cross-attention。
BEVFormer (2022)：BEV 时序自注意力，开启 BEV 序列时序建模。
BEVDet4D (2022)：相邻 BEV 特征 concat，简单有效的两帧时序。
PETRv2 (2022)：隐式 3D PE 的帧级时序对齐，只用相邻两帧（BEV/帧级、object 之外的载体）。
SOLOFusion (2022)：长时序 BEV 记忆，把帧数堆到 16+1。
Sparse4D / DETR4D (2022)：稀疏 query × 多帧透视特征，精度高但重复计算多。
StreamPETR (2023，本篇)：object-centric 时序——query 作 RNN 隐状态、memory queue 逐帧递归传播 + MLN 运动补偿，长时序低开销、在线 streaming，首个在线相机方法达到 LiDAR 可比精度。
后续 Sparse4Dv2/v3、StreamPETR 衍生：把「query 递归传播」进一步发扬，成为稀疏时序 3D 检测的主流范式。
一句话定位：PETRv2 是「帧级 BEV 时序对齐」，StreamPETR 把时序载体从 BEV/帧迁移到 object query，用递归传播换来长时序 + 低开销 + 动目标友好。

9.3 流式视频训练 + 梯度截断（省显存关键）

Petr3D.forward_train 把一个长序列拆成「前段不回传梯度 + 后段回传」，对应附录 A.2「detach 前 6 帧、只对最后 2 帧算梯度和 loss」：
def forward_train(self, img_metas=None, ..., **data):
    T = data['img'].size(1)
    prev_img = data['img'][:, :-self.num_frame_backbone_grads]   # 历史帧
    rec_img  = data['img'][:, -self.num_frame_backbone_grads:]   # 最近帧(回传梯度)
    rec_img_feats = self.extract_feat(rec_img, self.num_frame_backbone_grads)
    if T - self.num_frame_backbone_grads > 0:
        self.eval()                            # 历史帧用 eval 模式
        with torch.no_grad():                  # ★ 历史帧 backbone 不回传梯度
            prev_img_feats = self.extract_feat(prev_img, T-self.num_frame_backbone_grads, True)
        self.train()
        data['img_feats'] = torch.cat([prev_img_feats, rec_img_feats], dim=1)
    else:
        data['img_feats'] = rec_img_feats
    losses = self.obtain_history_memory(...)   # 逐帧前向, 只对最后几帧 return_losses
    return losses
obtain_history_memory 里逐帧循环：i >= num_nograd_frames 才 requires_grad、i >= num_grad_losses 才算 loss。这样长序列训练时，只有最后 1~2 帧真正回传梯度，显存占用大幅下降，但 memory queue 仍然把前面所有帧的时序信息递归累积了进来——这是「用很长的序列训练却不爆显存」的核心技巧。

10. 局限性与未来方向

远距离假阳性（Failure cases）：如 Fig. 6 红圈所示，StreamPETR 在 30m 范围内的密集物体上表现优秀，但在远处物体上有较多 False Positive。这是相机方法的通病——远处深度估计本就困难。论文认为在复杂城市场景里，远处的重复预测影响相对可控。
运动物体仍逊于静态物体：即便有 MLN 和时序传播，运动物体的检测精度仍明显落后静态物体，说明动目标建模还有提升空间。
依赖准确 ego-pose：MLN 的运动补偿建立在 ego-pose 矩阵之上，pose 噪声会直接影响历史 query 的对齐质量。
DETR3D 扩展增益有限：把 object-centric 套到 DETR3D（局部稀疏注意力）上提升不如 PETR 范式大，说明该范式更适合全局注意力的检测器。
未来方向：把长序列建模思想推广到更多任务（论文最后明确希望给社区在「长序列建模」上提供新思路），以及与深度/占据等更密集表征的结合。

11. 总结

StreamPETR 提出了一个高效的长序列 3D 检测框架，核心是 object-centric 时序范式：用稀疏 object query 当时序隐状态，通过 memory queue 逐帧递归传播长时序信息，再用 motion-aware layer normalization（MLN）隐式编码自车与物体运动。它在引入可忽略的存储和计算开销下取得领先性能，是首个在线多视角相机方法达到与 LiDAR 方法可比的精度（nuScenes test 67.6% NDS / 65.3% AMOTA）。
放到 PETR 谱系里看，它把时序载体从 PETRv2 的「BEV/帧级对齐」迁移到「object query 递归传播」，一举解决了 BEV 时序「动目标建模弱」和透视时序「长时序算力贵」两个老问题，成为后续稀疏时序 3D 检测（Sparse4Dv2/v3 等）的范式起点。
核心创新一句话各表：
Object-centric 时序：query 作 RNN 隐状态，memory queue 逐帧 FIFO 递归传播，长时序低开销。
MLN：条件 LayerNorm 隐式编码并解耦自车 + 物体运动，单 ego-pose 编码就 +2.0% mAP。
Hybrid attention：当前 query ⊕ 历史 memory query 一次注意力同时做时序交互 + 去重，几乎零开销。
流式训练 + 工程：梯度截断（只回传最后 2 帧）+ Flash Attention + DN，让长序列训练又稳又省显存。
