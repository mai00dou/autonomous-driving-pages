1. 研究背景与动机

1.1 DETR 是个革命性的设计

2020 年 ECCV 上 Carion 等人提出 DETR，第一次把目标检测做成纯端到端的范式：
去掉了 anchor、NMS、人工分配规则这些「手工部件」
用 Transformer 编解码器 + 一组可学习的 object query，靠 Hungarian Match 直接学到 set-to-set 的预测
训练目标干净，推理也无需任何后处理
但 DETR 上线之后业界发现两个致命问题：

问题

表现

收敛极慢

COCO 上要 500 epoch 才能收敛，是 Faster R-CNN 的 10–20 倍

小目标差

只有 20.5，比 Faster R-CNN+FPN 的 26.6 低一截
Deformable DETR 这篇就是奔着解决这两个问题来的。

1.2 DETR 慢和差的根源：全局 attention 用错了地方

把 Transformer 直接套到图像 feature map 上，会撞上两个本质问题：
(1) Attention 初始化时几乎是均匀的。
对一个 query，初始化时 attention weight 接近 （ 是 key 数量）。要让模型学会聚焦到稀疏的有用位置，需要长时间 epoch 去把这些权重「挤」到几个关键 token 上。这就是 500 epoch 的来源。
(2) Encoder 的全图 self-attention 复杂度是 。
对一张 feature map 来说，token 数 = ，每个 query 都要 attend 所有 个 key。这导致：
高分辨率特征用不起 → 小目标的细节没了
直接接 FPN 多尺度也不行：把 拼起来 token 数翻几倍，显存直接爆

1.3 Deformable Conv 的启发

DETR 的两个痛点，2017 年的 Deformable Convolution（DCN，Dai et al.）其实给了答案——稀疏采样：每个输出位置只看 个学出来的偏移点，不看全图。它的优点：
对前景敏感（采样点会自动聚到目标边缘和内部）
复杂度跟 线性相关，跟 无关
可学的 offset 让它比固定卷积感受野灵活得多
DCN 唯一缺的就是 element-pair 关系建模能力（self-attention 的核心优势）。
Deformable DETR 的思路就是：把这两个东西揉一起——让 attention 不再看全图，只看每个 query 周围学出来的 个采样点。既保留了 Transformer 的 set 关系建模，又拿到 DCN 的稀疏 + 多尺度自由组合优势。

2. 核心问题分析

把 DETR 的痛点摊开来讲，其实是三件事：

2.1 收敛慢的本质：attention 是「学出来」的稀疏，不是「设计出来」的稀疏

对一张 的 feature map，初始化时 query 对所有 key 的 attention 几乎是均匀分布：

模型要先学会把这堆几乎相等的权重「挤」到几个有用的位置上，再学怎么组合它们。500 epoch 主要消耗在前面这个「学稀疏」上。
Deformable Conv 给出的反例：直接设计上就是只看 个点（ ），不需要先用 attention 把全图过滤一遍，省了一大段优化路径。

2.2 小目标差的本质：高分辨率特征用不起

小目标天然要在高分辨率（小 stride）feature map 上检测——比如 stride=8 的 。但 DETR 的全图 self-attention 复杂度是：

stride 减半 → token 数 4 倍 → 显存和算力 16 倍。所以 DETR 没办法接 FPN，只用 stride=32 的 ，小目标就废了。DETR-DC5 用空洞卷积把 stride 从 32 降到 16，AP_S 从 20.5 涨到 22.5，但 GPU hours 直接从 2000 飙到 7000，性价比极差。

2.3 多尺度集成困难

主流检测器（FPN/PAN/BiFPN）都靠多尺度融合提点，但 DETR 用全局 attention 把所有 level 拼起来：
token 数 （4 个 level）
每个 query 都要看完所有 level 所有 token
显存代价不可接受
业界其实试过把 FPN 套到 DETR 上，性能反而没明显提升（见 §7 Table 2 BiFPN 一行）—— 这暗示 DETR 的瓶颈不在「特征不够多尺度」，而在 attention 本身没办法稀疏 + 多尺度联合工作。

2.4 总结：要换的是 attention 模块本身

DETR 的整体范式（CNN backbone + Transformer + object query + Hungarian Match + set 预测）是没问题的。要改的是「Transformer attention 在 feature map 上的工作方式」——这是 Deformable DETR 选定的切入点：attention 模块从全图改成稀疏可学采样。

3. 技术架构详解

3.1 整体架构总览

Deformable DETR 沿用 DETR 的「CNN backbone + Transformer encoder-decoder + object query + Hungarian set prediction」骨架，只换掉了 attention 模块：encoder 的 self-attention、decoder 的 cross-attention 全部改成 multi-scale deformable attention。decoder 的 self-attention（query 之间）保持原版 nn.MultiheadAttention 不变。
整体数据流：
ResNet 输出 三层多尺度特征
每层 conv 投影到 通道，再用 stride=2 conv 在 上叠一个 ，凑成 个 level
4 个 level flatten + concat 成一长串 token，加 sinusoidal 位置编码 + 可学的 scale-level embedding
进 6 层 Deformable Encoder：每层 self-attention（每个像素作为 query 自己）+ FFN
6 层 Deformable Decoder 消费 encoder memory：每层 standard self-attn 对 object query 做交互 + multi-scale deformable cross-attn 从 memory 取信息 + FFN
每层 decoder 输出都过 class head + bbox head，与 GT 做 Hungarian Match，所有层都加 aux loss

3.4 Deformable Encoder

把 DETR 的 vanilla self-attention 直接换成 multi-scale deformable self-attention，其余结构（FFN、residual、LayerNorm）不变：
输入和输出都是同分辨率的 4 级特征：encoder 不下采样，只做 token 间信息交换
每个 query 像素的 reference point = 它自己（在 4 个 level 上的对应归一化坐标）
scale-level embedding  ：和 sinusoidal 位置编码一起加到 token 上，让模型分得清「这个 token 来自哪个 level」。  是可学的，初始化为 normal
encoder 参数在 4 个 level 间共享（一组   矩阵，所有 level 都用）
每层 encoder layer 的计算：
src2 = MSDeformAttn(query=src+pos, ref=每像素自身, value=src, ...)
src  = LN(src + Dropout(src2))     # 残差 + LN
src  = src + Dropout(FFN(src))     # FFN: Linear(256→1024) → ReLU → Linear(1024→256)
src  = LN(src)

3.5 Deformable Decoder

decoder layer 比 encoder 多一个 standard self-attention（让 object query 之间互相 talk），cross-attention 才是 deformable 的：
# 1. self-attention：标准的 nn.MultiheadAttention（query 之间）
q = k = tgt + query_pos
tgt = LN(tgt + self_attn(q, k, tgt))
# 2. cross-attention：MSDeformAttn 从 encoder memory 取信息
tgt2 = MSDeformAttn(
    query        = tgt + query_pos,
    ref_points   = sigmoid(MLP(query_pos))    # 学出来的 2-d 坐标
    value        = encoder_memory,
    ...)
tgt = LN(tgt + Dropout(tgt2))
# 3. FFN
tgt = LN(tgt + Dropout(FFN(tgt)))
为什么 self-attention 不用 deformable？
deformable attention 是 query→feature map 上的 key 设计的（key 必须有空间位置）；object query 之间没有空间结构，标准 attention 更合适。
每个 object query 的 reference point   由 query embedding 经一层 Linear + sigmoid 学出来——这等于让每个 query 提前选定自己感兴趣的图像位置，DETR 里每个 query 是一个抽象的「slot」，没有显式空间锚定，这是慢收敛的另一个原因。Deformable 给了它一个具体的「我先盯着哪儿」。

3.6 Iterative Bounding Box Refinement

DETR 让每个 decoder 层独立预测 box；Deformable DETR 改成 每层修正上一层的预测，思路类似光流估计里的 RAFT iterative refinement：

实现要点：
每层 decoder 有独立的 bbox_embed（不共享参数）
第 层 cross-attention 用第 层预测的 box 中心 作为新 reference point；sampling offset 还按 box 大小 缩放，让采样点贴着上一层预测的框
训练稳定性技巧：梯度只反传 ，不反传 （detach），防止链式累积导致的训练崩塌
初始 box: ，作者实验过 0.05 / 0.2 / 0.5 都差不多
带 refinement 后 AP 从 43.8 → 45.4（+1.6），是个白送的提升。

3.7 Two-Stage Deformable DETR

DETR 原版的 object query 跟当前图像无关——纯随机初始化的 embedding。Deformable DETR 利用 deformable attention 的高效性，干脆把 encoder 当 RPN 用：
Stage 1（encoder-only RPN）：
encoder 输出后，每个像素都作 anchor（论文里说 "each pixel is assigned as an object query"）
加一个 3-layer FFN 预测
加一个 Linear 预测 binary class（前景/背景）
初始 box 大小用 （ ）—— 不同 level 给不同初始尺度
top-K 个高分的预测当作 region proposal（K = 300，等于 num_queries），不做 NMS
Stage 2（decoder + iterative refine）：
把 stage-1 的 top-300 proposal 当作 decoder 的初始 reference points（4-d box 而不是 2-d 中心）
proposal 坐标过 sinusoidal pos embed + Linear 得到 query_embed 和 tgt
后面就是普通的 6 层 decoder + iterative box refinement
带 two-stage 后 AP 从 45.4 → 46.2（+0.8）。

4.6 Sampling Offset 的初始化（关键工程技巧）

MSDeformAttn._reset_parameters() 里这段是收敛快的另一个秘密：
constant_(self.sampling_offsets.weight.data, 0.)         # 权重置零
thetas = torch.arange(M) * (2π / M)                        # M=8 个均匀角度
grid_init = stack([cos(θ), sin(θ)], -1)                    # 8 个单位向量
grid_init = grid_init / grid_init.abs().max(-1)            # 归一化到 ±1 方框
grid_init = grid_init.view(M, 1, 1, 2).repeat(1, L, K, 1)
for i in range(K):
    grid_init[:, :, i, :] *= (i + 1)                       # K 个半径 1, 2, 3, 4
self.sampling_offsets.bias = grid_init.view(-1)
效果：初始化时，8 个 head × K=4 个采样点排成 8 个方向 × 4 圈半径的星形阵列，覆盖 ref-point 周围一圈区域。这相当于给模型一个合理的「先验感受野」，再在训练中学着微调，比从全零或 Xavier 随机初始化收敛快很多。

▼ grad_attn_weight 经 softmax 反传 → attention_weights.weight 的梯度
▼ grad_value 反传到上层 input_flatten → encoder/backbone
注意 iterative refinement 里 reference_points = new_reference_points.detach() 这行——它把跨层的 box 修正梯度截断了，每层的 Δb 只通过自己那层的 head 反传，不会沿着 ref-point 链一路堆到 stage-1 的 box，避免了梯度爆炸。

6. 核心创新点

6.1 创新点 1：Deformable Attention（稀疏 K 采样替代全图 attention）

这是文章的根基。把 attention 的 key 集合从 降到 ，复杂度从 降到 。代价：理论上有些重要的远程信息可能 miss，但靠 学出来的 offset 自动找重要点 弥补——本质是一个 query-conditioned sparse attention。

6.2 创新点 2：Multi-Scale Feature Aggregation 不靠 FPN

Multi-scale deformable attention 在 个 level 上同时采样 + 加权，softmax 跨 level 归一化。一个 attention 就把跨 level 信息融了，FPN/BiFPN 都成了多余的（Table 2 实验验证）。
意义：
给 DETR 系列开启了多尺度特征的大门
大目标自动 attend low-resolution，小目标自动 attend high-resolution，学出来比手工 FPN 路径更灵活
节省了 FPN 的额外卷积成本

6.3 创新点 3：Iterative Bounding Box Refinement

每层 decoder 修正上一层的 box 预测，类似光流估计的 RAFT。带来两个好处：
AP +1.6（从 43.8 → 45.4）几乎免费
decoder 每一层都是 box-aware 的，sampling 点直接贴着 box 取，而不是固定围着初始 ref-point
工程上用 detach() 切断跨层 box 梯度，是训练稳定性的关键。

6.4 创新点 4：Two-Stage Deformable DETR

让 encoder 自己当 RPN，每个像素都作为 anchor 预测一个 box。top-K（K=300）当作 region proposal 喂 decoder——这等于让 object query 第一次有了「图像感知」的初始化，不再是抽象 slot。
带来 AP +0.8（45.4 → 46.2），更重要的是它启发了后面 DAB-DETR、DINO 的 anchor-based query 范式。

6.5 创新点 5：Box-Aware Reference Point + Sampling Offset 调制

两个对应的工程 trick，让 attention 学出来跟 box 强耦合：
reference point 用归一化坐标，跨 level 自动 rescale
sampling offset 按 box 大小调制： ，让大目标采样范围大、小目标采样范围小
sampling offset 的星形初始化：8 head × 4 半径，给模型一个合理的「初始感受野先验」，不用从随机起步

9.2 MSDeformAttn 模块（models/ops/modules/ms_deform_attn.py）

class MSDeformAttn(nn.Module):
    def __init__(self, d_model=256, n_levels=4, n_heads=8, n_points=4):
        # 4 个 Linear 是核心
        self.sampling_offsets = nn.Linear(d_model, n_heads * n_levels * n_points * 2)  # 256→256
        self.attention_weights = nn.Linear(d_model, n_heads * n_levels * n_points)     # 256→128
        self.value_proj = nn.Linear(d_model, d_model)
        self.output_proj = nn.Linear(d_model, d_model)
        self.im2col_step = 64                                                          # CUDA 分块大小
    def _reset_parameters(self):
        # 关键：sampling_offsets 的 bias 初始化成 8 head × K 半径的星形
        constant_(self.sampling_offsets.weight.data, 0.)
        thetas = torch.arange(self.n_heads) * (2π / self.n_heads)   # 8 个角度
        grid_init = stack([cos(θ), sin(θ)], -1)
        grid_init = grid_init / grid_init.abs().max(-1)              # 归一化到 ±1
        grid_init = grid_init.view(M, 1, 1, 2).repeat(1, L, K, 1)
        for i in range(K):
            grid_init[:, :, i, :] *= (i + 1)                         # 半径 1, 2, 3, 4
        self.sampling_offsets.bias = nn.Parameter(grid_init.view(-1))
        # attention_weights 全零（softmax 后均匀）
        constant_(self.attention_weights.weight.data, 0.)
        constant_(self.attention_weights.bias.data, 0.)
        # value_proj、output_proj 用 Xavier

9.3 forward 关键步骤

def forward(self, query, reference_points, input_flatten, spatial_shapes,
            level_start_index, padding_mask=None):
    # 1. value 投影 + 切 head
    value = self.value_proj(input_flatten)
    value = value.masked_fill(padding_mask[..., None], 0)
    value = value.view(N, Len_in, self.n_heads, self.d_model // self.n_heads)
        # [B, ΣHℓWℓ, M=8, C/M=32]
    # 2. 从 query 出 offset 和 weight
    sampling_offsets = self.sampling_offsets(query).view(
        N, Len_q, M, L, K, 2)                                # [B, Nq, 8, 4, 4, 2]
    attention_weights = self.attention_weights(query).view(
        N, Len_q, M, L * K)                                  # [B, Nq, 8, 16]
    attention_weights = F.softmax(attention_weights, -1).view(N, Len_q, M, L, K)
        # softmax 跨 (L, K) 一起做
    # 3. 把 ref-point + offset 转成具体采样坐标
    if reference_points.shape[-1] == 2:
        # 单层情形：normalize offset 到归一化坐标
        offset_normalizer = stack([spatial_shapes[..., 1], spatial_shapes[..., 0]], -1)
        sampling_locations = reference_points[:, :, None, :, None, :] \
                           + sampling_offsets / offset_normalizer[None, None, None, :, None, :]
    elif reference_points.shape[-1] == 4:
        # iterative refinement / two-stage 情形：用 box w/h 调制 offset
        sampling_locations = reference_points[:, :, None, :, None, :2] \
                           + sampling_offsets / self.n_points * reference_points[:, :, None, :, None, 2:] * 0.5
    # 4. 调 CUDA kernel 一把把 bilinear+加权融合做完
    output = MSDeformAttnFunction.apply(
        value, spatial_shapes, level_start_index,
        sampling_locations, attention_weights, self.im2col_step)
    return self.output_proj(output)

9.4 PyTorch autograd Function（models/ops/functions/ms_deform_attn_func.py）

class MSDeformAttnFunction(Function):
    @staticmethod
    def forward(ctx, value, spatial_shapes, level_start_index,
                sampling_locations, attention_weights, im2col_step):
        ctx.im2col_step = im2col_step
        output = MSDA.ms_deform_attn_forward(...)              # 调 CUDA
        ctx.save_for_backward(value, spatial_shapes, level_start_index,
                              sampling_locations, attention_weights)
        return output
    @staticmethod
    @once_differentiable
    def backward(ctx, grad_output):
        # 一次性反传 3 个梯度（value, sampling_loc, attn_weight）
        grad_value, grad_sampling_loc, grad_attn_weight = MSDA.ms_deform_attn_backward(...)
        return grad_value, None, None, grad_sampling_loc, grad_attn_weight, None
@once_differentiable 表示这个 backward 不允许二阶求导（节省内存，detection 任务用不上 Hessian）。

9.5 CUDA kernel 设计（ms_deform_attn_cuda.cu + ms_deform_im2col_cuda.cuh）

forward 入口签名：
at::Tensor ms_deform_attn_cuda_forward(
    const at::Tensor &value,                 // [B, ΣHℓWℓ, M, C/M]
    const at::Tensor &spatial_shapes,        // [L, 2]
    const at::Tensor &level_start_index,     // [L]
    const at::Tensor &sampling_loc,          // [B, Nq, M, L, K, 2]
    const at::Tensor &attn_weight,           // [B, Nq, M, L, K]
    const int im2col_step                    // 默认 64
);
实现要点：
im2col 思路：把 sample + 加权当成一个 "im2col" 操作，每个 query × head × level × point 取一次值再加权累加
batch 切块：im2col_step = min(batch, 64)，避免一次开太大显存
bilinear 内联：在 CUDA kernel 内手写 4 个 corner 的双线性插值，不调用 grid_sample
backward 同样手写 col2im：grad_value、grad_sampling_loc、grad_attn_weight 一次 kernel 全算出来
CPU 实现直接 AT_ERROR("Not implement on cpu")——这模块基本上只能 GPU 跑。

def ms_deform_attn_core_pytorch(value, value_spatial_shapes, sampling_locations, attention_weights):

    value_list = value.split([H * W for H, W in value_spatial_shapes], dim=1)
    sampling_grids = 2 * sampling_locations - 1
    sampling_value_list = []
    for lid_, (H_, W_) in enumerate(value_spatial_shapes):
        value_l_ = value_list[lid_].flatten(2).transpose(1, 2).reshape(N*M, C/M, H_, W_)
        sampling_grid_l_ = sampling_grids[:, :, :, lid_].transpose(1, 2).flatten(0, 1)

                                          align_corners=False)
        sampling_value_list.append(sampling_value_l_)
    attention_weights = attention_weights.transpose(1, 2).reshape(N*M, 1, Lq, L*P)

9.7 Iterative Box Refinement 的工程实现（deformable_transformer.py）

class DeformableTransformerDecoder(nn.Module):
    def forward(self, tgt, reference_points, src, ...):
        for lid, layer in enumerate(self.layers):
            # 关键：reference_points 是 4-d 还是 2-d 决定不同的处理
            if reference_points.shape[-1] == 4:
                reference_points_input = reference_points[:, :, None] \
                    * torch.cat([src_valid_ratios, src_valid_ratios], -1)[:, None]
            else:
                reference_points_input = reference_points[:, :, None] * src_valid_ratios[:, None]

output = layer(output, query_pos, reference_points_input, src, ...)

            # iterative bounding box refinement
            if self.bbox_embed is not None:
                tmp = self.bbox_embed[lid](output)
                if reference_points.shape[-1] == 4:
                    new_reference_points = tmp + inverse_sigmoid(reference_points)
                    new_reference_points = new_reference_points.sigmoid()
                else:
                    new_reference_points = tmp
                    new_reference_points[..., :2] = tmp[..., :2] + inverse_sigmoid(reference_points)
                    new_reference_points = new_reference_points.sigmoid()
                reference_points = new_reference_points.detach()  # 关键：截断梯度

9.8 Two-Stage 实现关键代码

if self.two_stage:
    # 1. encoder 输出做 proposal
    output_memory, output_proposals = self.gen_encoder_output_proposals(
        memory, mask_flatten, spatial_shapes)
    # 2. 用 decoder 最后一个 head 当 RPN head（hack：第 num_layers 个）
    enc_outputs_class = self.decoder.class_embed[self.decoder.num_layers](output_memory)
    enc_outputs_coord_unact = self.decoder.bbox_embed[self.decoder.num_layers](output_memory) \
                            + output_proposals
    # 3. top-K
    topk = self.two_stage_num_proposals     # 300
    topk_proposals = torch.topk(enc_outputs_class[..., 0], topk, dim=1)[1]
    topk_coords_unact = torch.gather(enc_outputs_coord_unact, 1,
                                     topk_proposals.unsqueeze(-1).repeat(1, 1, 4))
    topk_coords_unact = topk_coords_unact.detach()         # 关键：截断梯度
    reference_points = topk_coords_unact.sigmoid()
    pos_trans_out = self.pos_trans_norm(self.pos_trans(
        self.get_proposal_pos_embed(topk_coords_unact)))
    query_embed, tgt = torch.split(pos_trans_out, c, dim=2)
get_proposal_pos_embed 用 sinusoidal 把 4-d box 转成 2C 维 pos embed，再 split 成 query_embed (256) + tgt (256)。

9.10 注意事项

d_model 必须是 head 数 ×2 的幂：每 head 维度 = 256/8 = 32 是 2 的幂，CUDA kernel 用了 unrolling 优化
num_levels 默认 4：设其他值仓库能跑，但 CUDA 没特别优化
im2col_step：默认 64，太小会增加 kernel 调用次数，太大会爆显存。如果 batch 小 < 64 自动用 batch
第一次跑前要 cd models/ops && sh make.sh 编译 CUDA 扩展，否则 import MSDeformAttn 会失败

10. 局限性与未来方向

10.1 局限性

(1) K 是固定数，不能 query-adaptive
每个 query 不管图像内容怎样，永远采 个点。理想情况下应该让 query 自己决定要采几个点（小目标少、大目标多），目前是固定值。
(2) decoder cross-attention 本质是 1-to-LK 而不是 1-to-N
每个 object query 只看 个采样位置，远没有覆盖全图。这意味着：
如果某个真正重要的 token 不在 ref-point 周围，模型就 miss 了
训练初期 ref-point 还没学好的时候，可能采到错误位置导致难以修正
拥挤场景下两个目标的 ref-point 太接近，sampling 会重叠
(3) Reference point 是显式预测的，不像 anchor box 那么稳
DETR 的 query 抽象，Deformable DETR 的 query 用 ref-point 锚定，但 ref-point 只是 2-d 中心点，没有 size 信息。DAB-DETR 后来直接把 query 升级成 4-d anchor box 是这个的自然延伸。
(4) Two-stage 的 stage-1 没法精细分类
stage-1 RPN 只做 binary 前景/背景，stage-2 才做多类别。对类别多的场景（COCO 80 类）没问题，但对 1000+ 类长尾场景效率不够。
(5) 推理慢于纯卷积
Deformable DETR 的 19 FPS 比 Faster R-CNN+FPN 的 26 FPS 慢 27%——主要是 unordered memory access（每个 query 采的位置都不一样，cache 不友好）。这是稀疏 attention 共有的代价。
(6) 端到端但不真正"无 NMS"
虽然不用 NMS，但 Hungarian Match 的代价是训练每个 batch 都要解一个 LSAP（线性指派），CPU 开销不小，大 batch 时是个瓶颈。

10.2 未来方向（论文之后已实现的）

(1) 更结构化的 query → DAB-DETR / DN-DETR / DINO
把 query 升成 4-d anchor box（DAB），加 GT 噪声去噪训练（DN），混合 query selection（DINO）—— 这条线最终把 DETR 系列推到了 SOTA。
(2) 把 deformable attention 推到 3D
BEVFormer / UniAD / OccFormer 等自动驾驶工作直接复用：reference point 升到 3D（BEV 网格点），sampling offset 也成 3D，bilinear → trilinear。
(3) 时空 deformable attention
BEVFormer 的 temporal self-attention 在时间维度也用 deformable，串联多帧 BEV 特征。马爷的 OCC 项目里 5 帧 1500ms 的 temporal fusion 跟这个思路一脉相承。
(4) 跨模态 deformable cross-attention
PETR 系列、UniAD 把 deformable attention 用作多视角图像 → 3D query 的 cross-attention，把多视角特征聚合到 BEV/3D query 上。
(5) 更高效的 sparse attention 变体
Sparse4D、StreamPETR 等进一步把采样点数压到 K=8 或 K=4 搞 streaming inference。

11. 总结

Deformable DETR 是一篇用一个简单想法解决一个根本问题的好工作：
观察：DETR 慢、小目标差，本质是「全图 attention 不适合 image feature map」
想法：从 deformable conv 借稀疏采样，从 transformer 留 query-conditioned weighted sum，揉成一个新 attention
结果：10× 收敛加速、AP_S 从 20.5 → 28.8、46.2 AP（vs DETR 42.0）
影响：成为后续 DAB-DETR / DN-DETR / DINO 系列的 base，更外推到 BEVFormer / UniAD / OccFormer 等 3D 自动驾驶基建
它的技术核心可以一句话概括：
每个 query 不再 attend 全图，而是 attend ref-point 周围学出来的 K 个稀疏点，跨多尺度统一加权。
工程上有几个关键 trick 是落地的关键：
CUDA im2col kernel 把 sample + 加权融合，避免显存爆炸
sampling offset 的星形初始化给模型合理的初始感受野
iterative box refinement 用 detach() 截断跨层梯度，训练稳定
two-stage 让 encoder 自己当 RPN，给 query 注入图像感知初始化
对马爷的 E2E Occupancy 项目，这篇的启发主要是 §4 的复杂度设计哲学（稀疏 K 采样 vs 全图 attention 的取舍），以及 §3.6 的 iterative refinement 思路（每层 decoder 修正 ref-point）—— 这两个直接关系到 OCC head 设计。
