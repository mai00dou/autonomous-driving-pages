在原论文中，CenterPoint的流程大概如上图所示：在第一阶段（如图1中的a,b,c)，使用CenterPoint检测三维目标的检测框中心点，并回归其检测框大小，方向和速度。在第二阶段（如图1中的d)设计了一个refinement模块，对于第一阶段中的检测框，使用检测框中心的点特征回归检测框的score并进行refinement。
在OpenPCDet中，由于专注于检测模块而缺少了追踪模块，这一Pipeline得到进一步的简化。这里主要介绍一下CenterPoint这个经典的网络在OpenPCDet这个框架中的实现方法。这里以VoxelNet为3DBackbone，并且以ResNet作为2D特征提取器的结构为例：

CenterPoint在OpenPCDet中的实现
在OpenPCDet中，这一流程主要分为以下三个模块：
Backbone_3D，主要对三维点云进行特征提取，通常使用VoxelNet或者Pointpillar的形式。
Backbone_2D，将提取的3D特征处理成为BEV下的特征图，并进一步对BEV下的特征进一步进行特征提取。
DenseHead，生成heatmap，并回归出3D Boundingbox的中心高度，长宽高和航向角等信息。
代码使用的是nuscenes数据集，接下来将分别进行介绍:
2.代码详解

2.1 MeanVFE

这一块的主要作用就是对Voxel的特征进行平均。每个voxel内有十个位置，但其实并不是每个位置都有有效的点。而传入的参数num_point_features中存储了每个voxel的有效点的数量，因此便于每个voxel对特征作均值。

class MeanVFE(VFETemplate):def __init__(self, model_cfg, num_point_features, **kwargs):super().__init__(model_cfg=model_cfg)self.num_point_features = num_point_featuresdef get_output_feature_dim(self):return self.num_point_featuresdef forward(self, batch_dict, **kwargs):"""
        Args:
            batch_dict:
                voxels: (num_voxels, max_points_per_voxel, C)
                voxel_num_points: optional (num_voxels)
            **kwargs:
        Returns:
            vfe_features: (num_voxels, C)
        """# voxel_features （12000， 10， 5） voxel_num_points （12000）# 5个通道应该是 （z, y, x ,ring_index, intensity）voxel_features, voxel_num_points = batch_dict['voxels'], batch_dict['voxel_num_points']points_mean = voxel_features[:, :, :].sum(dim=1, keepdim=False) #求出每个体素内 10 个点的sumnormalizer = torch.clamp_min(voxel_num_points.view(-1, 1), min=1.0).type_as(voxel_features)points_mean = points_mean / normalizer #求出每个体素内 sum / 点数batch_dict['voxel_features'] = points_mean.contiguous() # points_mean（12000， 5）return batch_dict

2.2 VoxelRes_8x

这一步是真正地对3D信息进行特征提取，其输入是上一步经过VFEMean之后的VFE特征，输出为经过稀疏卷积编码后的特征向量。
具体来看，输入参数有两个。1.经过VFEMean之后的VFE特征(num_voxels, C)；2.描述每个voxel坐标位置的voxel_coords (num_voxels, 4), [batch_idx, z_idx, y_idx, x_idx]。由于代码太多，这里重点关注 VoxelResBackBone8x 这个类下的forward函数：

class VoxelResBackBone8x(nn.Module):def __init__(self, model_cfg, input_channels, grid_size, **kwargs):super().__init__()def forward(self, batch_dict):"""
        Args:
            batch_dict:
                batch_size: int
                vfe_features: (num_voxels, C)
                voxel_coords: (num_voxels, 4), [batch_idx, z_idx, y_idx, x_idx]
        Returns:
            batch_dict:
                encoded_spconv_tensor: sparse tensor
        """# voxel_features（12000，5）：Voxel特征均值，   voxel_coords（12000， 4） ：Voxel坐标的索引voxel_features, voxel_coords = batch_dict['voxel_features'], batch_dict['voxel_coords']batch_size = batch_dict['batch_size']# 真正的变成了体素# 对voxel_features按照coors进行索引，coors在之前的处理中加入例如batch这个位置，变成了四维# 输出是一个（batch_size，channels, sparse_shap）的数据（batch_size， chanels， grid_nums_z， grid_nums_y， grid_nums_x）# 就是让数据按照coors里的坐标进行了排列，成为了标准的体素空间input_sp_tensor = spconv.SparseConvTensor(features=voxel_features,indices=voxel_coords.int(),spatial_shape=self.sparse_shape,batch_size=batch_size)x = self.conv_input(input_sp_tensor) #经过一个 indice_key='subm1' 的稀疏巻积 features.shape ([120000, 5]) -> ([120000, 16])x_conv1 = self.conv1(x) #([120000, 16]) -> ([120000, 16])x_conv2 = self.conv2(x_conv1) #  ([120000, 16]) -> ([113293, 32])x_conv3 = self.conv3(x_conv2) # ([55250, 64])x_conv4 = self.conv4(x_conv3)  # ([20650, 128])#随着稀疏巻积的进行，spatial size会发生改变，特征维度也会改变# for detection head# [200, 176, 5] -> [200, 176, 2]out = self.conv_out(x_conv4) #这里用的巻积形式是 SparseConv3d 而不是 SubMConv3dbatch_dict.update({'encoded_spconv_tensor': out,'encoded_spconv_tensor_stride': 8})return batch_dict
在forward函数中，主要做了两件事：
1.对voxel_features按照coors进行索引，使其真正变成了体素的形式的特征，其维度为（batch_size， chanels， grid_nums_z， grid_nums_y， grid_nums_x）。
2.将体素形式的特征输入一个以稀疏卷积为卷积方式的类似于ResNet结构的Backbone，提取点云的稀疏特征。

2.3 Height_compression

这一步的主要目的是将提取出的点云稀疏特征转换到二维的BEV视角上。事实上这个转换过程非常简单粗暴。首先把稀疏特征转换成为体素特征的格式，然后把Dense维度和Chanel维度合并，便成为了BEV视角上的2D特征。

class HeightCompression(nn.Module):def __init__(self, model_cfg, **kwargs):super().__init__()self.model_cfg = model_cfgself.num_bev_features = self.model_cfg.NUM_BEV_FEATURESdef forward(self, batch_dict):"""
        Args:
            batch_dict:
                encoded_spconv_tensor: sparse tensor
        Returns:
            batch_dict:
                spatial_features:
        """encoded_spconv_tensor = batch_dict['encoded_spconv_tensor']spatial_features = encoded_spconv_tensor.dense() #从 spcov的格式中取出最终的按照体素格式特征的tensor N, C, D, H, W = spatial_features.shape #（batch_size, channel, Dense, Height, Width）spatial_features = spatial_features.view(N, C * D, H, W) #合着整个HeightCompression就是 换一个 view, 把chanel 和 Dense合到一起batch_dict['spatial_features'] = spatial_featuresbatch_dict['spatial_features_stride'] = batch_dict['encoded_spconv_tensor_stride']return batch_dict

2.4 BaseBEVBackbone

将上一步中得到的2D特征进行进一步的特征提取。在代码中，上一步采集到的特征被命名为“spatial_features”。代码构建了一个self.blocks,其中blocks内部包含了以2D卷积+BN层+ReLU的多个串联的layer，此外还包含了上采样层，主要目的是为了增加不同尺度的特征。

class BaseBEVBackbone(nn.Module):def __init__(self, model_cfg, input_channels):super().__init__()   
def forward(self, data_dict):"""
        Args:
            data_dict:
                spatial_features : (batch, c, W, H)
        Returns:

            CenterPoint中一共存在两个下采样分支，
            分支一: (batch_size, Chanel * D, H , W)
            分支二: (batch_size, Chanel * D, H , W)
            """x = self.blocks[i](x)stride = int(spatial_features.shape[2] / x.shape[2])ret_dict['spatial_features_%dx' % stride] = x# 如果存在deconv，则对经过conv的结果进行反卷积操作"""
            CenterPoint 中存在两个下采样，则分别对两个下采样分支进行反卷积操作
            """if len(self.deblocks) > 0:ups.append(self.deblocks[i](x))else:ups.append(x)# 将上采样结果在通道维度拼接if len(ups) > 1:"""

2.5 CenterHead

前面铺垫了这么多，终于可以进入CenterPoint的检测head了。在得到2D的BEV下的特征之后，CneterPoint的预测基本和CenterNet的流程差不多，只不过在代码中，heatmap的真值是在这一阶段生成的，而不是像CenterNet的代码一样在生成label的时候就生成heatmap的真值。
在nusenes数据集中，目标的类别一共被分为了6个大类，网络给每一个类都分配了一个head，装在headlist中，而每个head内部都为预测的参数，包括（center_x,y,z、dim、rot、vel ）分配了一个MLP，从而完成检测头的设计。
在assign_targets这个函数中，利用真值生成了基于高斯分布的heatmap,即基于指定的iou阈值生成gaussian radius，然后在范围内生成heatmap,与CenterNet没什么区别。
如果你对高斯圆和高斯半径是怎么确定的还不是很清楚，强烈建议看一下这位大佬的讲解视频:

2.6 Loss

Loss方面没有什么新意，由于对生成的特征图 H x W的每一个位置都进行了预测，与真值的shape相同。最终loss就是直接进行Focal Loss和L1 Loss即可。
3.总结
CenterPoint入选了2021的CVPR，从代码上看，其实创新点乏善可陈，但是其在工业界上确实非常受欢迎的方法之一，其简单的网络结构和能够与多种Backbone结合的灵活性，让其受到很多点云算法工作者的青睐。
最后，为了方便大家研究，我也把我标注的代码分享在这里，配置好OpenPCDet的环境之后直接跑即可。如果只是单纯的瞟一眼代码，其实还是很难理解其中每一步的实现过程。只有将环境搭建好真正调试起来，才能更好地理解代码中每一步的用意。
