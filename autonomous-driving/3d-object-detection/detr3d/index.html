DETR3D：3D-to-2D Queries 多视角 3D 检测（PETR 系起点）
论文：DETR3D: 3D Object Detection from Multi-view Images via 3D-to-2D Queries
作者：Yue Wang, Vitor Guizilini, Tianyuan Zhang, Yilun Wang, Hang Zhao, Justin Solomon
机构：MIT + Toyota Research Institute + CMU + Li Auto + 清华
会议：CoRL 2021
arXiv：2110.06922v1（2021-10-13）
Code：https://github.com/WangYueFt/detr3d
解读对象：PDF 主体 12 页 + 官方仓库 projects/mmdet3d_plugin（detr3d.py / detr3d_head.py / detr3d_transformer.py / hungarian_assigner_3d.py）
一句话定位：DETR3D 第一个把多相机 3D 检测做成「3D set-to-set 预测」——不预测稠密深度、不做 NMS，而是用一组稀疏 3D object query 解码出 reference point，再把这个 3D 点投影回各相机做 2D 特征采样来 refine query。它是 reference-point lift 范式的开山之作，PETR/PETRv2/StreamPETR/Sparse4D 全系都建立在它的 query 设计之上。这篇是整条 DETR→PETR 谱系正式进入 3D 的起点。

1. 研究背景与动机

1.1 低成本自动驾驶的 3D 检测难题

从纯视觉信息做 3D 目标检测，是低成本自动驾驶系统的长期挑战。LiDAR 点云天然带有可见物体的 3D 结构信息，而纯相机设置更加病态（ill-posed）：必须仅从 RGB 图像里的 2D 信息生成 3D 包围盒预测。

1.2 已有两条路线及其缺陷

论文把当时的方法归为两类，都不理想：
第一类，纯 2D 计算路线（CenterNet、FCOS3D）。直接套用为 2D 任务设计的检测管线来预测物体位姿和速度等 3D 信息，完全不考虑 3D 场景结构与传感器配置。多相机时必须独立处理每张图，再用 per-image NMS + global NMS 去冗余、跨相机融合——这是一组非并行化操作，带来显著推理开销，效率与效果之间存在陡峭的权衡。
第二类，伪 LiDAR / 深度重建路线。先用深度预测网络（如 PackNet）从图像重建出 pseudo-LiDAR 点云或 range 输入，再当成 3D 传感器数据跑 3D 检测。问题在于复合误差（compounding error）：深度估计不准会强烈拖累后续 3D 检测，而深度网络本身又容易过拟合训练分布、泛化差。

1.3 DETR3D 的主张：在 3D 空间里直接操作

DETR3D 提出在 2D 观测和 3D 预测之间做一个更优雅的过渡，不依赖稠密深度预测模块。核心思路是 top-down：
从一组跨数据集共享、端到端学习的稀疏 object 先验出发；
每个先验解码出一个 3D reference point，**反向投影（back-project）**到每个相机，用相机变换矩阵取回对应的 2D 图像特征；
取回的特征经多头自注意力交互，逐层 refine；
每层都读出包围盒参数，用 DETR 式的 set-to-set 损失训练。
这样既不做点云重建、也不做显式深度预测，对深度误差鲁棒；且无需 NMS 等后处理。在 nuScenes 上，不用 NMS 的 DETR3D 与用 NMS 的同期方法相当，在相机重叠区域显著领先。

1.4 四大贡献

提出从 RGB 图像做 3D 检测的精简模型，在每一层计算中就融合所有相机视图（而非最后阶段才合并），是首个把多相机检测建模为 3D set-to-set 预测的工作。
提出一个通过反向几何投影连接 2D 特征提取与 3D 框预测的模块，不受二级深度网络误差影响，并能天然利用多相机信息。
类似 Object DGCNN，无需 per-image / global NMS，与基于 NMS 的方法持平，在重叠区域大幅领先。

2. 核心问题分析

2.1 为什么「自顶向下」优于「自底向上」

论文把现有相机检测头归纳为 bottom-up：每张图预测稠密框 → 图间过滤冗余 → 后处理跨相机聚合。它有两个致命缺陷：
稠密框预测需要准确的深度感知，而这本身就是难题；
NMS 式的去冗余 + 聚合是非并行化操作，引入显著推理开销。
DETR3D 反过来用 top-down：从稀疏 3D query 出发，主动去图像里「取」需要的特征，把「2D 计算」和「3D 计算」交替进行，从根上绕开稠密深度和 NMS。

2.2 关键洞察：用相机几何做「3D→2D」的桥

DETR3D 的精髓是方向：不是从 2D 像素抬升到 3D（lift），而是先有 3D 假设点（reference point），用已知的相机投影矩阵 lidar2img 把它打到各个相机平面，按需采样该位置的 2D 特征。深度信息隐含在「这个 3D 点投到不同相机的不同位置」这一几何约束里，由网络端到端学出来，而不是显式回归一张深度图。这一点是后续 PETR「把 3D 位置编码进 feature」思路的直接对话对象（DETR3D 把 3D 信息绑在 query 侧，PETR 反过来绑在 feature 侧——这是谱系里最重要的一次范式分叉，详见 §8）。
下面这张总览图给出完整数据流：多视角图像经 ResNet+FPN 编码，object query 解码出 3D reference point，投影回图像空间采样特征来 refine query，最后做 per-query 预测与 set-to-set 损失。

3.1 2D 特征提取（Feature Learning）

输入图像集合 （环视相机拍摄），相机矩阵 ，真值框 （每个框含 BEV 下的位置、尺寸、朝向角、速度），类别 。模型不使用点云。
图像用 ResNet + FPN 编码成 4 组特征 ，每组 对应 6 张图的某一级特征。多尺度特征为识别不同大小的物体提供丰富信息。
实现细节：ResNet-101 在第 3、4 阶段用可变形卷积（DCN）；FPN 输出 4 张特征图，尺寸为输入的 1/8、1/16、1/32、1/64；隐藏维 。

3.2 检测头：迭代式 3D-to-2D 采样（核心贡献）

用相机变换矩阵把这些中心投影到所有特征图；
用双线性插值采样特征，融合进 object query；
用多头自注意力描述 object 间的交互。

6. 核心创新点

6.1 范式创新：3D set-to-set 预测（首创）

DETR3D 是第一个把多相机 3D 检测建成 3D set-to-set 预测的工作。与「最后阶段才合并各相机预测」不同，它在每一层计算就融合所有相机视图——一个 3D reference point 同时投到所有相机采样，天然完成跨视图融合，重叠区域不再需要 NMS 去重。

6.2 机制创新：反向几何投影（backward projection）

不预测稠密深度、不重建点云，而是「3D 假设点 → 投影回 2D → 按需采样」。深度被几何约束隐式编码：一个 3D 点投到不同相机的不同 2D 位置，这组对应关系由 lidar2img 矩阵确定，网络只需学会用采到的特征精修 3D 位置。这从根上避免了 pseudo-LiDAR 的复合误差。

6.3 工程创新：iterative box refinement + 梯度截断

源码 Detr3DTransformerDecoder 里，每层用回归分支更新 reference point：
new_reference_points[..., :2] = tmp[..., :2] + inverse_sigmoid(reference_points[..., :2])
new_reference_points[..., 2:3] = tmp[..., 4:5] + inverse_sigmoid(reference_points[..., 2:3])
new_reference_points = new_reference_points.sigmoid()
reference_points = new_reference_points.detach() # 关键：截断跨层梯度
detach() 让每层只对「本层的精修量」负责，不把梯度传到上一层的 reference point——这与 Deformable DETR 的 iterative bbox refine 同款，是训练稳定的关键。Table 5 量化了层数收益：NDS 从第 0 层的 0.380 单调升到第 5 层的 0.425。

6.4 NMS-free + 深度-free 的双重解耦

DETR3D 同时甩掉了两个传统包袱：NMS（靠 set-to-set 的一对一匹配天然去重）和显式深度预测（靠反向投影几何）。这让推理高度并行、对深度误差鲁棒。代价是 mATE（平移误差）偏高——深度仍是这个问题的核心难点（§10）。

7. 实验结果与分析

7.1 数据集与设置

nuScenes：1000 个序列，每段约 20s、20 fps；每个样本含 6 相机 [前左、前、前右、后左、后、后右]，有完整内外参。每 0.5s 标注一次，训练/验证/测试分别 28k / 6k / 6k 样本，10 个类别参与评测。训练用 AdamW，weight decay ，初始 lr （第 8、11 epoch 衰减），共 12 epoch，8× RTX 3090（per-GPU batch=1），约 18 小时，推理不用任何 NMS。

8. 技术发展时间线（DETR3D 在 DETR→PETR 谱系里的位置）

DETR3D 是整条「DETR→PETR」谱系正式进入 3D 检测的第一块基石，理解它等于拿到后续全系的钥匙。

2020.05 DETR（Carion et al., ECCV 2020）：set prediction + 匈牙利匹配，去掉 anchor 和 NMS。DETR3D 的 set-to-set 损失、object query 概念直接继承自此。

2020.10 Deformable DETR（ICLR 2021）：稀疏 deformable attention + reference point + iterative box refinement。DETR3D 的 reference-point 设计、detach() 截断跨层梯度、inverse_sigmoid 精修，全部沿用自 Deformable DETR。

2021.10 DETR3D（本篇，CoRL 2021）：把 reference point 从 2D 抬到 3D，用相机投影矩阵做「3D→2D」反向采样，首次实现多相机 3D set-to-set 检测。确立了「3D 信息绑在 query 侧」的范式。

2022.05 PETR（ECCV 2022）：反转 DETR3D 的方向——不再让 query 主动投影采样，而是把 3D 位置编码（3D PE）烘进 image feature，让 decoder 退化为标准 transformer。这是谱系里最重要的一次范式分叉（feature 侧 vs query 侧）。

2022.06 PETRv2 / 2023.03 StreamPETR / 2023.06 Sparse4D / 2023.08 Far3D / SparseBEV：在 PETR/DETR3D 的 query + reference point 底座上，逐步加入时序、稀疏 4D 锚点、远距离长尾优化。

一句话：DETR3D = Deformable DETR 的 reference point 在 3D 世界的投射。它和 PETR 共同定义了 vision-only 多视角 3D 检测的两种 query 范式。

reference_points_cam = (reference_points_cam - 0.5) * 2              # → [-1,1] grid
sampled_feat = F.grid_sample(feat, reference_points_cam_lvl)        # 双线性采样
要点：用 lidar2img（内参×外参）一步到位投影；mask 用 z>eps 排除相机后方的点，再叠加 u,v∈[-1,1] 排除视野外的点。
③ 加权聚合（Detr3DCrossAtten.forward）
attention_weights = self.attention_weights(query).view(bs,1,num_query,num_cams,num_points,num_levels)
attention_weights = attention_weights.sigmoid() * mask               # 视锥外权重清零
output = (output * attention_weights).sum(-1).sum(-1).sum(-1)         # 对 cam/point/level 求和
pos_feat = self.position_encoder(inverse_sigmoid(reference_points_3d))# 3D 位置编码注入
return self.dropout(output) + inp_residual + pos_feat
要点：attention_weights 用 sigmoid 而非 softmax（各采样点独立加权，不竞争）；constant_init(attention_weights, val=0) 让训练初期权重均匀。
④ iterative refine + 梯度截断（Detr3DTransformerDecoder）
new_reference_points[...,:2] = tmp[...,:2] + inverse_sigmoid(reference_points[...,:2])
new_reference_points[...,2:3] = tmp[...,4:5] + inverse_sigmoid(reference_points[...,2:3])
reference_points = new_reference_points.sigmoid().detach()           # detach 是稳定关键
⑤ 匹配与损失（hungarian_assigner_3d.py / detr3d_head.py）：Hungarian 匹配，focal(cls) + L1(box)，每层都算 loss（d0.loss_cls…），code_weights[...,0.2,0.2] 给速度降权。

10. 局限性与未来方向

论文坦诚的局限：
单点投影感受野有限：每个 query 只采 reference point 一个位置的特征，receptive field 偏小。作者建议为每个 query 采样多个点（这正是 PETR 后续 / Sparse4D 多点采样的动机）。
平移误差（mATE）偏高：虽避免了显式深度，但深度估计仍是核心难题，远距离物体易漏检（Figure 3 失败案例）。
未用其它模态：检测头 input-agnostic，未来可融合 LiDAR/RADAR 提升鲁棒性。
可推广到室内导航、物体操纵等领域。

11. 总结

DETR3D 用一个新范式回应「从 2D 图像恢复 3D 信息」这一病态逆问题：不做 2D 纯计算、也不靠额外深度网络重建场景，而是在 3D 空间操作、用反向投影按需取图像特征。两大收益：(1) 去掉了中间表示（深度图/点云）这一复合误差来源；(2) 通过把同一 3D 点投到所有相机来天然利用多相机信息。它是 reference-point lift 范式的开山之作，也是马爷这条 DETR→PETR 谱系正式进入 3D 检测的起点。
对 E2E Occupancy 项目的启发：
DETR3D 的「3D query 投影回多相机采样」与项目里 11 视图特征融合的诉求高度同构——可作为稀疏 query 融合多视图的轻量替代（相比稠密 BEV attention 省显存）。
code_weights 给速度降权、detach() 截断跨层梯度，是直接可复用的训练稳定工程常数。
DETR3D「单点采样感受野有限」的局限，正是项目设计 occupancy query 采样策略时要规避的——应一开始就上多点/可变形采样。
