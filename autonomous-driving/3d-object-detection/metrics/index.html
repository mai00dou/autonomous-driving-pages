3D目标检测评估指标体系

1. mAP (mean Average Precision) - 平均精度均值

定义: 衡量检测器在不同类别上的整体检测准确率
公式:

AP = ∫₀¹ P(R) dR

mAP = (1/C) ∑ᶜᵢ₌₁ APᵢ
其中:
P(R): Precision-Recall曲线
C: 类别数量
APᵢ: 第i个类别的AP值
nuScenes特殊计算:
使用中心距离阈值: [0.5m, 1m, 2m, 4m]
对每个阈值计算AP，然后平均
只考虑2D中心距离匹配（不考虑高度）
侧重点:
检测召回率: 能找到多少真实目标
检测精确率: 检测结果中有多少是正确的
类别平衡性: 各类别检测性能是否均衡
您的结果: mAP = 0.2828 (28.28%)
说明检测召回率较低，可能存在漏检问题
建议检查小目标检测和远距离目标检测能力

2. mATE (mean Average Translation Error) - 平均位置误差

定义: 预测框中心与真实框中心的欧氏距离误差
公式:

ATE = √[(x_pred - x_gt)² + (y_pred - y_gt)² + (z_pred - z_gt)²]

mATE = (1/N) ∑ᴺᵢ₌₁ ATEᵢ
其中:
(x_pred, y_pred, z_pred): 预测中心坐标
(x_gt, y_gt, z_gt): 真实中心坐标
N: 匹配的检测框数量
侧重点:
定位精度: 3D空间中的位置估计准确性
深度估计: 特别是z轴（高度）的准确性
BEV定位: xy平面的定位能力
您的结果: mATE = 0.7734m
平均位置误差约77cm，属于中等水平
对于自动驾驶场景，建议优化到0.5m以下
可能需要改进深度估计网络或BEV特征提取

3. mASE (mean Average Scale Error) - 平均尺寸误差

定义: 预测框尺寸与真实框尺寸的相对误差
公式:

ASE = (1/3) ∑ᵈ∈{w,l,h} (1 - IoU₁ᴰ(d_pred, d_gt))

IoU₁ᴰ(a, b) = min(a,b) / max(a,b)

mASE = (1/N) ∑ᴺᵢ₌₁ ASEᵢ
其中:
w, l, h: 宽度、长度、高度
IoU₁ᴰ: 一维IoU（重叠比例）
侧重点:
尺寸估计: 目标长宽高的准确性
类别特征: 不同类别的尺寸先验是否合理
尺度不变性: 对不同大小目标的适应能力
您的结果: mASE = 0.2884
尺寸误差约29%，表现较好
说明模型对目标尺寸的估计相对准确
可能使用了有效的尺寸先验或回归策略

4. mAOE (mean Average Orientation Error) - 平均朝向误差

定义: 预测朝向角与真实朝向角的最小夹角误差
公式:

AOE = min(|θ_pred - θ_gt|, 2π - |θ_pred - θ_gt|)

mAOE = (1/N) ∑ᴺᵢ₌₁ AOEᵢ
其中:
θ_pred: 预测朝向角（弧度）
θ_gt: 真实朝向角（弧度）
取最小角度差（考虑周期性）
侧重点:
朝向估计: 目标行驶方向的准确性
角度回归: 处理角度周期性的能力
运动预测: 对后续轨迹预测的影响
您的结果: mAOE = 0.6976 rad (约40°)
朝向误差较大，需要重点优化
建议使用多bin分类+残差回归
或采用向量表示法（sin/cos编码）

5. mAVE (mean Average Velocity Error) - 平均速度误差

定义: 预测速度与真实速度的L2范数误差
公式:

AVE = √[(vx_pred - vx_gt)² + (vy_pred - vy_gt)²]

mAVE = (1/N) ∑ᴺᵢ₌₁ AVEᵢ
其中:
(vx_pred, vy_pred): 预测速度（m/s）
(vx_gt, vy_gt): 真实速度（m/s）
只考虑水平面速度（不含vz）
侧重点:
运动估计: 目标运动状态的准确性
时序建模: 利用多帧信息的能力
轨迹预测: 对规划模块的输入质量
您的结果: mAVE = 0.8637 m/s
速度误差约0.86m/s，偏高
建议引入时序模块（LSTM/Transformer）
或使用多帧融合提升运动估计

6. mAAE (mean Average Attribute Error) - 平均属性误差

定义: 属性分类错误率（1 - 准确率）
公式:

AAE = 1 - Accuracy_attr

Accuracy_attr = (正确分类的属性数) / (总属性数)

mAAE = (1/N) ∑ᴺᵢ₌₁ AAEᵢ
属性类别（nuScenes）:
车辆: vehicle.moving, vehicle.stopped, vehicle.parked
行人: pedestrian.moving, pedestrian.standing, pedestrian.sitting_lying_down
自行车: cycle.with_rider, cycle.without_rider
侧重点:
状态识别: 目标运动/静止状态判断
细粒度分类: 更精细的语义理解
场景理解: 对交通场景的深层理解
您的结果: mAAE = 0.2908
属性错误率约29%，中等水平
说明状态识别能力有待提升
建议增强时序特征或引入上下文信息

7. NDS (nuScenes Detection Score) - nuScenes综合检测分数

定义: 综合所有指标的加权评分
公式:

NDS = (1/10) × [5×mAP + ∑ᵀₚ₌₁ min(1, mTPₚ)]

其中 TP (True Positive) 指标包括:
- mATE: 转换为 1 - min(1, mATE)
- mASE: 转换为 1 - min(1, mASE)
- mAOE: 转换为 1 - min(1, mAOE/π)
- mAVE: 转换为 1 - min(1, mAVE)
- mAAE: 转换为 1 - min(1, mAAE)
完整计算:
NDS = (1/10) × [
    5 × mAP +
    (1 - min(1, mATE)) +
    (1 - min(1, mASE)) +
    (1 - min(1, mAOE/π)) +
    (1 - min(1, mAVE)) +
    (1 - min(1, mAAE))
]
权重分析:
mAP占50%权重（最重要）
其他5个指标各占10%权重
侧重点:
综合性能: 平衡检测和回归质量
实用性: 更接近实际应用需求
鲁棒性: 避免单一指标过拟合
您的结果: NDS = 0.3500
综合得分35%，有较大提升空间
主要受mAP和mAVE拖累
建议优先优化检测召回率和速度估计
